Monthly (Long) #2 · Avril 2026 · Devs Edition

🎯 Le fil de cette édition

Avril 2026 marque un tournant dans l'autonomie de GitHub Copilot pour les développeurs. Le Copilot cloud agent s'affranchit de l'ouverture automatique d'une pull request : il peut désormais mener une recherche approfondie, proposer un plan d'implémentation et itérer sur une branche avant que vous décidiez. La commande /fleet dans Copilot CLI pousse cette logique plus loin, en distribuant le travail entre plusieurs agents exécutés en parallèle. Le SDK Copilot sort en preview publique et Visual Studio reçoit une mise à jour majeure centrée sur l'extensibilité. Le message est clair : Copilot évolue vers un partenaire de développement autonome et transparent.

⚠️ Attention

  • GPT-5.1 Codex, GPT-5.1-Codex-Max et GPT-5.1-Codex-Mini dépréciés — Action required — Ces modèles ont été dépréciés le 1er avril 2026. Migrez dès maintenant vers GPT-5.3-Codex via vos paramètres de modèles Copilot.

🏆 Top 8 des mises à jour du mois

1. Copilot cloud agent : recherche, planification et itération sans PR obligatoire

Le Copilot cloud agent ne se limite plus aux workflows pull request. Il peut désormais mener une recherche approfondie sur votre codebase, proposer un plan d'implémentation pour validation, puis itérer sur une branche jusqu'à ce que vous décidiez d'ouvrir une PR.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. /fleet dans Copilot CLI : plusieurs agents en parallèle

La commande /fleet permet à Copilot CLI de décomposer une tâche en sous-tâches indépendantes et de les exécuter simultanément via des agents en arrière-plan. Idéal pour les refactors multi-fichiers, la génération de documentation ou les implémentations transversales.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

3. SDK Copilot en preview publique

Le SDK GitHub Copilot est disponible en preview publique pour Node.js/TypeScript, Python, Go, .NET et Java. Il expose le même runtime agent que Copilot CLI et cloud agent : invocation d'outils, streaming, sessions multi-tours, support OpenTelemetry et BYOK inclus.

→ Changelog: Copilot SDK in public preview

4. GitHub Copilot dans Visual Studio : mise à jour de mars

Visual Studio 2026 franchit un cap en extensibilité : agents personnalisés via des fichiers .agent.md, skills réutilisables, outil find_symbol avec navigation orientée language server, profilage de tests avec analyse CPU automatique, et correction de vulnérabilités NuGet directement depuis l'Explorateur de solutions.

→ Changelog: GitHub Copilot in Visual Studio — March update

5. Copilot cloud agent : commits signés et résolution de conflits de fusion

Le Copilot cloud agent signe désormais ses commits (label Verified), le rendant compatible avec les dépôts imposant "Require signed commits". Il peut aussi résoudre des conflits de fusion à la demande via un simple @copilot Merge in main and resolve the conflicts.

Sources:
→ Changelog: Copilot cloud agent signs its commits
→ Changelog: Ask @copilot to resolve merge conflicts on pull requests

6. Instructions personnalisées d'organisation pour Copilot (GA)

Les instructions d'organisation personnalisées passent en disponibilité générale. Les administrateurs Copilot Business et Enterprise définissent des instructions appliquées uniformément à Copilot Chat sur github.com, aux revues de code et au cloud agent – sans configuration repo par repo.

→ Changelog: Copilot organization custom instructions are generally available

7. Créer des issues GitHub depuis Slack avec Copilot

Mentionnez @GitHub dans n'importe quel canal Slack pour créer des issues GitHub en langage naturel, avec titre, corps, assignés, labels et jalons. Les sous-issues et le mode conversation pour itérer avant validation sont également supportés.

→ Changelog: Create issues from Slack with Copilot

8. GitHub Mobile : refonte du tab Copilot et gestion native des sessions d'agent

GitHub Mobile intègre un onglet Copilot repensé avec vue d'ensemble des sessions d'agent, filtres par état et mises à jour en temps réel. Logs de session natifs, création de PR depuis une session terminée et arrêt d'une session en cours sont désormais disponibles sur iOS et Android.

Sources:
→ Changelog: GitHub Mobile: Stay in flow with a refreshed Copilot tab and native session logs
→ Changelog: GitHub Mobile: Faster, more flexible agent assignment from issues
