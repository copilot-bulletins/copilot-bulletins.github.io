Monthly (Long) #1 · Mars 2026 · Devs Edition

🎯 Le fil de cette édition

Ce mois de mars marque un tournant agentique pour GitHub Copilot. L'ensemble des IDE — VS Code, JetBrains, Visual Studio — converge vers un modèle d'agent programmable, extensible via des hooks et des serveurs MCP. En parallèle, l'arrivée de GPT-5.4 et la refonte agentique du code review renforcent la qualité des interactions. Le coding agent, désormais intégré à Jira et capable d'exécuter les workflows Actions sans approbation manuelle, franchit un seuil d'autonomie notable.

La tendance est claire : Copilot passe du statut d'assistant contextuel à celui de coéquipier autonome, capable de planifier, exécuter et mémoriser le contexte de vos projets.

⚠️ Attention

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline: 26 mars 2026 (Gemini 3 Pro) / 1er avril 2026 (GPT-5.1, GPT-5.1-Codex, GPT-5.1-Codex-Mini, GPT-5.1-Codex-Max) — Migrez vers Gemini 3.1 Pro ou GPT-5.3-Codex et vérifiez les modèles supportés.

🚀 Passez à l'action

  • Activez GPT-5.4 dans vos paramètres de modèles Copilot pour profiter du dernier modèle agentique d'OpenAI.
  • Configurez des hooks d'agent pour automatiser le linting, les tests ou les contrôles de sécurité dans vos sessions agent.
  • Demandez une code review Copilot depuis votre terminal avec gh pr edit --add-reviewer @copilot`` — en savoir plus.
  • Explorez Copilot Memory pour permettre à Copilot de retenir les conventions de votre dépôt.
  • Des questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 8 des mises à jour du mois

1. VS Code v1.110 : les agents franchissent un cap

Cette version introduit les hooks pour automatiser des actions clés pendant les sessions agent, la possibilité de forker une conversation, l'auto-approve via /yolo, les plugins d'agent et la compaction de contexte. Les suggestions Next Edit s'étendent désormais à l'ensemble du fichier.

→ Changelog: GitHub Copilot in Visual Studio Code v1.110 – February release

2. GPT-5.4 disponible dans GitHub Copilot

Le dernier modèle d'OpenAI, optimisé pour le coding agentique, est désormais disponible dans tous les IDE, sur github.com, sur Mobile et via le CLI. Les premiers tests montrent des taux de réussite record en raisonnement multi-étapes et exécution de tâches complexes.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

3. JetBrains : les capacités agentiques passent en GA

Les agents personnalisés, sub-agents et l'agent de planification sont maintenant GA. Les hooks d'agent arrivent en preview, avec l'auto-approve MCP et le support des fichiers d'instructions (AGENTS.md, CLAUDE.md). Un panneau de raisonnement étendu offre une visibilité sur la réflexion du modèle.

→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs

4. Code review : architecture agentique et intégration CLI

Le code review Copilot repose désormais sur une architecture agentique qui explore le contexte du dépôt pour des commentaires plus ciblés et moins de bruit. Depuis le terminal, gh pr edit --add-reviewer @copilot`` lance une review sans quitter votre workflow.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Changelog: Request Copilot code review from GitHub CLI

5. Copilot coding agent : Jira, autonomie et choix de modèle

L'agent de codage s'intègre désormais à Jira en preview publique : assignez un ticket et recevez une pull request. Vous pouvez aussi choisir le modèle via le sélecteur dans les commentaires de PR et désactiver l'approbation manuelle des workflows Actions.

Sources:
→ Changelog: GitHub Copilot coding agent for Jira is now in public preview
→ Changelog: Pick a model for @copilot in pull request comments
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

6. Serveur MCP Figma : du design au code et retour

Le serveur MCP Figma permet maintenant un workflow bidirectionnel dans VS Code : tirez du contexte design vers votre code, puis renvoyez votre UI rendue vers Figma sous forme de frames éditables. Un pont concret entre design et développement.

→ Changelog: Figma MCP server can now generate design layers from VS Code

7. Copilot Memory activée par défaut

Copilot Memory, désormais activée par défaut pour les utilisateurs Pro et Pro+, permet à Copilot de mémoriser les conventions et dépendances d'un dépôt. Ces connaissances sont partagées entre le coding agent, le code review et le CLI pour un contexte continu.

→ Changelog: Copilot Memory now on by default for Pro and Pro+ users in public preview

8. Visual Studio : mise à jour de février

Visual Studio 2026 enrichit Copilot avec l'acceptation partielle des suggestions par clic, la coloration syntaxique des completions, les mémoires persistantes, les prompts personnalisés en commandes slash, un serveur MCP NuGet intégré et un agent dédié WinForms.

→ Changelog: GitHub Copilot in Visual Studio — February update
