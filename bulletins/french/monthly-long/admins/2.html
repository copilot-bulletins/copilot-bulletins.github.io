Monthly (Long) #2 · Mars 2026 · Admins Edition

🎯 Le fil de cette édition

Ce mois de mars est marqué par la maturité organisationnelle du Copilot cloud agent. GitHub a livré une série de contrôles au niveau organisation — runners, pare-feu, instructions personnalisées — qui permettent aux administrateurs de déployer l'agent à grande échelle avec des règles cohérentes. Simultanément, les métriques d'utilisation ont atteint une couverture complète : CLI par utilisateur, suivi de l'activité de l'agent de code. Pour les équipes IT, c'est le mois où Copilot passe d'un outil à déployer à une plateforme à gouverner.

⚠️ Attention

  • Dépréciation prochaine de Claude Sonnet 4 dans GitHub Copilot — Date limite : 1er mai 2026 — Activez Claude Sonnet 4.6 dans les politiques de modèles Copilot de votre organisation avant la date limite.
  • Mise à jour de la Déclaration de confidentialité et des Conditions d'utilisation — Date limite : 24 avril 2026 — À partir du 24 avril, les données d'interaction des utilisateurs Copilot Free, Pro et Pro+ pourront être utilisées pour entraîner les modèles IA, sauf opposition. Les comptes Business et Enterprise ne sont pas concernés, mais informez vos utilisateurs de l'option de désinscription disponible dans les paramètres Copilot.

🚀 Passez à l'action

  • Configurez le runner par défaut du Copilot cloud agent et verrouillez-le pour vos dépôts depuis les paramètres Cloud agent de votre organisation.
  • Gérez le pare-feu de l'agent cloud au niveau organisation pour contrôler les accès réseau : consultez la documentation sur le pare-feu de l'agent.
  • Définissez des instructions personnalisées d'organisation pour guider Copilot dans vos dépôts via les paramètres Copilot.
  • Activez et consultez les métriques d'utilisation Copilot complètes via l'API Copilot usage metrics.
  • Des questions ou besoin d'assistance ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 7 des mises à jour du mois

1. Contrôles organisationnels centralisés pour le Copilot cloud agent

Les administrateurs peuvent désormais définir le runner par défaut de l'agent cloud, le verrouiller pour tous les dépôts, et gérer le pare-feu à l'échelle de l'organisation. Adieu la configuration dépôt par dépôt : GitHub centralise la gouvernance du Copilot cloud agent.

Sources : → Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

2. Les commits du Copilot cloud agent sont désormais signés

L'agent cloud signe chaque commit — ils apparaissent comme Verified sur GitHub. Il peut désormais fonctionner dans des dépôts protégés par la règle « Require signed commits », levant un blocage courant pour les équipes appliquant des politiques de sécurité strictes sur leurs branches.

→ Changelog: Copilot cloud agent signs its commits

3. Les instructions personnalisées d'organisation sont en disponibilité générale

Les instructions personnalisées d'organisation, en préversion depuis avril 2025, passent en GA. Les administrateurs Business et Enterprise peuvent définir des instructions qui guident Copilot dans tous les contextes : Chat sur GitHub.com, révision de code et agent cloud.

→ Changelog: Copilot organization custom instructions are generally available

4. Métriques d'utilisation Copilot : couverture complète de l'adoption

Les métriques couvrent désormais l'intégralité du spectre Copilot : activité CLI par utilisateur dans les rapports organisationnels, et identification des utilisateurs actifs du coding agent. Les administrateurs disposent d'une vue 360° pour piloter l'expansion et cibler les besoins d'activation.

Sources : → Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports
→ Changelog: Copilot usage metrics now identify active Copilot coding agent users

5. Copilot cloud agent : au-delà des pull requests

L'agent cloud ne se limite plus à l'ouverture de pull requests. Il peut désormais travailler sur une branche sans créer de PR, générer un plan d'implémentation pour validation préalable, ou mener des recherches approfondies dans le code. Les administrateurs qui activent l'agent offrent à leurs équipes un éventail d'usages bien plus riche.

→ Changelog: Research, plan, and code with Copilot cloud agent

6. SDK Copilot en préversion publique

Le SDK GitHub Copilot expose le même moteur d'exécution que l'agent cloud. Les équipes plateforme peuvent désormais intégrer les capacités agentiques de Copilot dans leurs propres outils et services, en Node.js, Python, Go, .NET ou Java — sans réécrire la couche d'orchestration IA.

→ Changelog: Copilot SDK in public preview

7. Gouvernance MCP dans Visual Studio et améliorations de l'intégration Jira

Visual Studio introduit la gouvernance MCP pour les entreprises : les administrateurs définissent une liste d'autorisation des serveurs MCP, contrôlant l'accès aux données sensibles. L'intégration Jira s'améliore avec la sélection de modèle directement depuis Jira et l'accès à Confluence via MCP.

Sources : → Changelog: GitHub Copilot in Visual Studio — March update
→ Changelog: GitHub Copilot for Jira — Public preview enhancements
