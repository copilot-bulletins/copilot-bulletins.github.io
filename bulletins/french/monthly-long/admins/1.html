Monthly (Long) #1 · Mars 2026 · Admins Edition

🎯 Le fil de cette édition

Ce mois-ci, GitHub franchit un cap décisif en matière de gouvernance de l'IA. Les Enterprise AI Controls et le plan de contrôle des agents passent en disponibilité générale, offrant aux administrateurs une visibilité sans précédent sur l'activité agentique. En parallèle, les métriques d'utilisation Copilot atteignent elles aussi la GA avec une couverture élargie. Combinées à l'arrivée de Claude et Codex sur les plans Business, et à la nouvelle architecture agentique de la revue de code, ces avancées renforcent considérablement la capacité des équipes IT à piloter le déploiement Copilot avec contrôle et confiance.

⚠️ Attention

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline : 26 mars (Gemini 3 Pro) / 1er avril (GPT-5.1) — Vérifiez vos politiques de modèles et activez les alternatives recommandées (Gemini 3.1 Pro, GPT-5.3-Codex) dans vos paramètres Copilot.

🚀 Passez à l'action

  • Explorez les nouveaux AI Controls dans les paramètres entreprise pour superviser l'activité agentique et configurer les politiques d'agents personnalisés.
  • Activez les métriques d'utilisation Copilot pour suivre l'adoption, y compris la télémétrie CLI et le mode plan.
  • Configurez la revue de code Copilot automatique sur vos dépôts ou votre organisation pour tirer parti de la nouvelle architecture agentique.
  • Gérez les règles d'exclusion de contenu via l'API REST pour automatiser la gouvernance à l'échelle.
  • Des questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 8 des mises à jour du mois

1. Enterprise AI Controls et plan de contrôle des agents en disponibilité générale

Les administrateurs disposent désormais d'une vue centralisée pour gérer les politiques IA, superviser les sessions agentiques en temps réel avec de nouveaux filtres (statut, dépôt, utilisateur), et consulter les journaux d'audit. Des rôles personnalisés permettent de déléguer l'administration IA sans accorder un accès admin complet.

Sources:
→ Changelog: Enterprise AI Controls & agent control plane now generally available
→ Changelog: Discover and manage agent activity with new session filters

2. Copilot Metrics en GA : tableaux de bord, APIs et télémétrie étendue

Les métriques d'utilisation Copilot sont maintenant en disponibilité générale avec des tableaux de bord à l'échelle entreprise et organisation, un support data residency, et des contrôles d'accès granulaires. La couverture s'étend à la télémétrie CLI (niveau entreprise et utilisateur), au mode plan, et corrige les identifiants EMU pour une cohérence API totale.

Sources:
→ Changelog: Copilot metrics is now generally available
→ Changelog: Copilot usage metrics now includes enterprise-level GitHub Copilot CLI activity
→ Changelog: Copilot usage metrics now includes user-level GitHub Copilot CLI activity
→ Changelog: Copilot metrics now includes plan mode

3. Claude et Codex disponibles pour Copilot Business et Pro

Claude (Anthropic) et OpenAI Codex sont désormais accessibles comme agents de codage pour les plans Business et Pro, sans abonnement supplémentaire. Les administrateurs doivent activer ces agents au niveau entreprise puis organisation. Les trois agents partagent la même plateforme avec gouvernance unifiée, mémoire partagée et journalisation d'audit centralisée.

→ Changelog: Claude and Codex now available for Copilot Business & Pro users

4. Revue de code Copilot : architecture agentique et 60 millions de revues

La revue de code Copilot repose désormais sur une architecture agentique avec appel d'outils, offrant des retours plus ciblés, moins de bruit et une compréhension contextuelle du dépôt. Les organisations utilisant des runners auto-hébergés doivent effectuer une configuration initiale. Plus de 12 000 organisations activent déjà la revue automatique sur chaque pull request.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Blog: 60 million Copilot code reviews and counting
→ Changelog: Request Copilot code review from GitHub CLI

5. GPT-5.4 disponible dans GitHub Copilot

Le dernier modèle agentique d'OpenAI est maintenant accessible sur tous les plans payants Copilot. Les administrateurs Enterprise et Business doivent activer la politique GPT-5.4 dans les paramètres Copilot avant que leurs utilisateurs puissent sélectionner ce modèle.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

6. Nouveaux contrôles pour le coding agent

Un nouveau paramètre de dépôt permet aux administrateurs de désactiver l'approbation humaine des workflows Actions déclenchés par le coding agent, accélérant la boucle de feedback au prix d'un risque de sécurité accru. Par ailleurs, la configuration réseau pour le coding agent utilise désormais le routage par abonnement — les équipes avec des runners auto-hébergés doivent mettre à jour leurs règles de pare-feu.

Sources:
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
→ Changelog: Network configuration changes for Copilot coding agent now in effect

7. API Content Exclusion et Copilot Memory

Les administrateurs peuvent désormais gérer les règles d'exclusion de contenu Copilot via une API REST (GET/SET) au niveau organisation et entreprise, facilitant l'automatisation de la gouvernance. En parallèle, Copilot Memory est activé par défaut pour les utilisateurs Pro/Pro+, avec un contrôle total laissé aux administrateurs enterprise via les politiques Copilot.

Sources:
→ Changelog: Copilot Content Exclusion REST API in public preview
→ Changelog: Copilot Memory now on by default for Pro and Pro+ users in public preview

8. Mises à jour majeures des IDE : VS Code, Visual Studio et JetBrains

Les releases de février apportent des capacités agentiques renforcées dans les trois IDE principaux : hooks d'agent pour appliquer des politiques, auto-approbation MCP configurable, agents personnalisés et sub-agents en GA dans JetBrains, fork de conversation et plugins d'agent dans VS Code, et mémoire Copilot dans Visual Studio. Les administrateurs Business/Enterprise doivent activer la politique « Editor preview features » pour certaines fonctions.

Sources:
→ Changelog: GitHub Copilot in Visual Studio Code v1.110 – February release
→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs
→ Changelog: GitHub Copilot in Visual Studio — February update
