🎯 Le fil de cette édition

Ces deux dernières semaines, Copilot franchit une nouvelle étape vers l'ubiquité : les sessions CLI deviennent pilotables à distance depuis le web ou le mobile, l'agent cloud intègre une phase de planification et de recherche en amont du code, et le CLI expérimente une revue croisée entre familles de modèles. L'assistant sort du bureau pour accompagner les développeurs dans tous leurs contextes.

⚠️ Attention

  • Enforcing new limits and retiring Opus 4.6 Fast from Copilot Pro+ — Action required — Le modèle Opus 4.6 Fast est retiré des plans Copilot Pro+ ; passez à Opus 4.6 comme alternative. De nouveaux plafonds de requêtes sont déployés progressivement : utilisez le mode Auto pour basculer automatiquement vers un autre modèle en cas de limite atteinte.

🚀 Passez à l'action

  • Essayez les sessions CLI à distance : lancez copilot --remote et ouvrez la session depuis n'importe quel appareil (documentation).
  • Activez le mode expérimental dans Copilot CLI avec /experimental pour accéder à Rubber Duck, le second modèle cross-family qui évalue vos plans.
  • Utilisez l'agent cloud Copilot pour planifier et itérer avant d'ouvrir une pull request.
  • Des questions ou besoin d'aide ? Contactez votre équipe GitHub, nous sommes là pour vous accompagner.

🏆 Top 5 des mises à jour

1. Pilotez vos sessions Copilot CLI depuis le web ou le mobile

La Copilot CLI s'ouvre sur le monde avec copilot --remote : votre session est diffusée en temps réel vers GitHub.com ou l'appli mobile (lien + QR code). Envoyez des instructions, approuvez des permissions et changez de mode à distance — parfait pour les longues tâches en arrière-plan.

→ Changelog: Remote control CLI sessions on web and mobile in public preview

2. Rubber Duck : une revue croisée entre modèles dans Copilot CLI

En mode expérimental, Copilot CLI introduit Rubber Duck : un second modèle d'une autre famille d'IA (GPT-5.4 si vous utilisez Claude) qui évalue les plans de l'agent avant l'exécution. Résultat : Claude Sonnet + Rubber Duck comble 74,7 % de l'écart de performance avec Opus seul. Activez via /experimental.

→ Blog: GitHub Copilot CLI combines model families for a second opinion

3. L'agent cloud Copilot passe en mode planification

Copilot cloud agent évolue : demandez un plan d'implémentation avant toute écriture de code, itérez sur une branche et ouvrez une PR seulement quand vous êtes prêt. Une session de recherche approfondie est aussi disponible pour explorer et comprendre le dépôt avant d'agir.

→ Changelog: Research, plan and code with Copilot cloud agent

4. Résolvez les conflits de fusion en trois clics

Plus besoin de résoudre manuellement les conflits de merge : un bouton "Fix with Copilot" apparaît désormais sur github.com dans les PRs conflictuelles. Cliquez, confirmez, et l'agent cloud prend la main — il résout les conflits, vérifie que les tests passent, et pousse les modifications.

→ Changelog: Fix merge conflicts in three clicks with Copilot cloud agent

5. Sélection de modèle pour les agents tiers Claude et Codex

Comme pour Copilot cloud agent, il est désormais possible de choisir le modèle lors du lancement d'une tâche avec les agents Claude (Anthropic) et Codex (OpenAI) sur github.com. Les modèles disponibles incluent Claude Sonnet 4.6, Claude Opus 4.6, GPT-5.2-Codex, GPT-5.3-Codex et GPT-5.4.

→ Changelog: Model selection for Claude and Codex agents on github.com
