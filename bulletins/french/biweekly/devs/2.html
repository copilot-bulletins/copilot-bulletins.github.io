Biweekly #2 · Du 23 mars au 6 avril 2026 · Devs Edition

🎯 Le fil de cette édition

Ces deux dernières semaines ont été dominées par une accélération de l'outillage agentique autour de GitHub Copilot : SDK public, parallélisation native dans le CLI, nouvelles capacités dans les IDEs. GitHub consolide une plateforme sur laquelle les développeurs peuvent non seulement utiliser des agents, mais aussi en construire, les orchestrer et les intégrer dans leurs propres workflows.

⚠️ Attention

  • GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated — Deadline: 1er avril 2026 — Ces trois modèles ont été retirés de toutes les expériences Copilot (Chat, complétion de code, mode agent). Migrez vers GPT-5.3-Codex via vos paramètres de modèles Copilot.

🚀 Passez à l'action

  • Explorez le SDK GitHub Copilot pour embarquer les capacités agentiques de Copilot dans vos propres applications (Node.js, Python, Go, .NET, Java).
  • Essayez /fleet dans Copilot CLI pour paralléliser vos tâches sur plusieurs fichiers simultanément — lancez copilot dans votre terminal et entrez /fleet (votre objectif).
  • Configurez les runners de l'agent cloud Copilot au niveau de l'organisation pour imposer vos propres runners (plus puissants ou self-hosted).
  • Mettez à jour vos intégrations qui référencent les modèles GPT-5.1-Codex en consultant la liste des modèles supportés.
  • Des questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 5 des mises à jour

1. Copilot SDK en préversion publique

Le SDK GitHub Copilot expose le même moteur d'agents qui propulse Copilot CLI et l'agent cloud. En cinq langages (Node.js, Python, Go, .NET, Java), il offre invocation d'outils, streaming, opérations sur fichiers et sessions multi-tours — sans orchestration à construire from scratch. Support BYOK (OpenAI, Azure AI Foundry, Anthropic) et OpenTelemetry inclus.

→ Changelog: Copilot SDK in public preview

2. /fleet : plusieurs agents en parallèle dans Copilot CLI

La commande /fleet permet à Copilot CLI d'orchestrer plusieurs sous-agents simultanément sur des fichiers indépendants. Un orchestrateur décompose votre objectif en tâches parallèles et séquentielles, les exécute en arrière-plan et synthétise les résultats. Idéal pour les refactorisations multi-fichiers, la génération de documentation ou les implémentations de features cross-cutting.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

3. GitHub Copilot dans Visual Studio — mise à jour de mars

La mise à jour de mars apporte les agents personnalisés (fichiers .agent.md dans le dépôt), les agent skills (instructions réutilisables), l'outil find_symbol pour la navigation symbolique dans le code (C++, C#, TypeScript, etc.), et le profilage de performances assisté par Copilot directement depuis le Test Explorer. Les serveurs MCP respectent désormais les politiques d'autorisation d'organisation.

→ Changelog: GitHub Copilot in Visual Studio — March update

4. L'agent cloud Copilot signe désormais ses commits

Chaque commit effectué par l'agent cloud Copilot apparaît maintenant avec le badge Verified sur GitHub. Cette amélioration débloque l'utilisation de l'agent dans les dépôts où la règle de protection « Require signed commits » est activée — un prérequis courant dans les environnements à compliance stricte.

→ Changelog: Copilot cloud agent signs its commits

5. Contrôles de runner au niveau de l'organisation pour l'agent cloud

Les admins d'organisation peuvent désormais définir un runner par défaut pour l'agent cloud Copilot (standard, large runner ou self-hosted) et empêcher les dépôts de le surcharger. Cela permet d'uniformiser les environnements d'exécution de l'agent à l'échelle de l'organisation — plus de performance, accès aux ressources internes, contrôle cohérent.

→ Changelog: Organization runner controls for Copilot cloud agent

📌 Également dans cette édition

  • Organization firewall settings for Copilot cloud agent — Les admins peuvent désormais gérer le pare-feu de l'agent cloud au niveau de l'organisation : activer/désactiver, gérer la liste blanche personnalisée, et contrôler les accès des dépôts enfants.
  • Copilot usage metrics: per-user CLI activity in org reports — Les rapports organisation incluent maintenant l'activité CLI par utilisateur (sessions, requêtes, tokens, version CLI) pour faciliter les déploiements et l'allocation des coûts.
  • Agent-driven development in Copilot Applied Science — Un retour d'expérience concret sur le développement agent-first : stratégies de prompting, architecture, et comment traiter Copilot comme un junior engineer.
