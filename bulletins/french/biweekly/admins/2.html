Biweekly #2 · Du 10 au 24 mars 2026 · Admins Edition

🎯 Le fil de cette édition

Ces deux semaines ont été marquées par un renforcement significatif des outils de gouvernance et de traçabilité de Copilot pour les administrateurs. Des métriques d'usage plus précises, des pistes d'audit sur les commits de l'agent, des contrôles de sécurité configurables et un nouveau programme de support à long terme pour les modèles reflètent une priorité claire : donner aux admins une visibilité complète sur l'activité IA dans leur organisation.

🏆 Top 5 des mises à jour

1. Les métriques d'usage résolvent désormais la sélection automatique de modèles

Les usages en mode "Auto" apparaissaient jusqu'ici sous une étiquette générique dans les rapports. Désormais, chaque requête est attribuée au modèle réellement utilisé — visible dans l'API REST et les tableaux de bord — pour une traçabilité précise indispensable aux workflows de conformité et d'audit à l'échelle de l'organisation.

→ Changelog: Copilot usage metrics now resolve auto model selection to actual models

2. GPT-5.3-Codex : premier modèle à support long terme pour les entreprises

GitHub lance son programme LTS pour les modèles Copilot : GPT-5.3-Codex sera garanti disponible pendant 12 mois (jusqu'au 4 février 2027), offrant la stabilité nécessaire aux révisions internes de sécurité. Il deviendra également le modèle de base par défaut pour Copilot Business et Enterprise le 17 mai 2026.

→ Changelog: GPT-5.3-Codex long-term support in GitHub Copilot

3. Configurez les outils de validation de l'agent de codage Copilot

Les administrateurs de dépôts peuvent désormais choisir quels outils de validation l'agent de codage exécute : CodeQL, GitHub Advisory Database, secret scanning et Copilot code review. Ces outils sont activés par défaut et gratuits, mais il est possible de désactiver certains checks — utile si CodeQL ralentit un projet spécifique.

→ Changelog: Configure Copilot coding agent's validation tools

4. Traçabilité complète des commits de l'agent de codage

Chaque commit produit par Copilot coding agent inclut désormais un trailer Agent-Logs-Url qui relie directement le commit aux logs de session complets. Ce lien permanent facilite l'audit du code généré par l'IA : les équipes peuvent comprendre pourquoi Copilot a effectué un changement sans perdre la trace de la session d'origine.

→ Changelog: Trace any Copilot coding agent commit to its session logs

5. Secret scanning dans les agents IA via le GitHub MCP Server

Le GitHub MCP Server peut désormais analyser les modifications de code à la recherche de secrets exposés avant chaque commit ou pull request. En préversion publique pour les dépôts avec GitHub Secret Protection activé, cette fonctionnalité s'intègre nativement dans les workflows des agents IA compatibles MCP.

→ Changelog: Secret scanning in AI coding agents via the GitHub MCP Server
