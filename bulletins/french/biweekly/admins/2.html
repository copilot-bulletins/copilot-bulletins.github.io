Biweekly #2 · Du 23 mars au 6 avril 2026 · Admins Edition

🎯 Le fil de cette édition

Cette quinzaine marque un tournant dans la maturité opérationnelle de Copilot pour les équipes d'administration : GitHub centralise les contrôles de gouvernance de l'agent cloud Copilot au niveau de l'organisation. Runners, pare-feu réseau, conformité des commits signés — les outils pour déployer Copilot à grande échelle, avec les bonnes guardrails, sont désormais entre les mains des administrateurs.

⚠️ Attention

  • GPT-5.1-Codex, GPT-5.1-Codex-Max et GPT-5.1-Codex-Mini dépréciés — Date limite : 1er avril 2026 (passée) — Migrez dès maintenant vos workflows vers GPT-5.3-Codex ; les administrateurs Copilot Enterprise doivent activer les modèles alternatifs via les politiques de modèles dans les paramètres Copilot.

🚀 Passez à l'action

  • Définissez un runner par défaut et verrouillez-le pour toute votre organisation dans les paramètres de l'agent cloud Copilot.
  • Gérez le pare-feu de l'agent Copilot à l'échelle de votre organisation via la documentation du pare-feu de l'agent.
  • Consultez l'activité CLI par utilisateur dans les métriques d'utilisation Copilot via l'API REST.
  • Activez les nouveaux modèles de substitution dans vos politiques Copilot d'entreprise pour remplacer les modèles GPT-5.1-Codex dépréciés.
  • Questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 5 des mises à jour

1. Contrôle des runners au niveau de l'organisation pour l'agent cloud Copilot

Les administrateurs peuvent désormais définir un runner par défaut pour l'agent cloud Copilot dans toutes les organisations et bloquer la personnalisation au niveau des dépôts. Idéal pour imposer des runners plus puissants ou des runners auto-hébergés donnant accès aux ressources internes.

→ Changelog: Organization runner controls for Copilot cloud agent

2. Paramètres du pare-feu organisationnel pour l'agent cloud Copilot

Les administrateurs disposent désormais d'un contrôle centralisé du pare-feu réseau de l'agent cloud Copilot : activation/désactivation globale, liste blanche recommandée, listes d'autorisation personnalisées au niveau de l'organisation, et restriction des modifications par les dépôts.

→ Changelog: Organization firewall settings for Copilot cloud agent

3. L'agent cloud Copilot signe désormais tous ses commits

Chaque commit produit par l'agent cloud Copilot est désormais signé et apparaît comme Vérifié sur GitHub. L'agent est maintenant compatible avec la règle de protection de branche « Require signed commits », levant ainsi un blocage majeur dans les environnements à conformité renforcée.

→ Changelog: Copilot cloud agent signs its commits

4. Métriques Copilot : activité CLI par utilisateur dans les rapports d'organisation

Les rapports d'organisation incluent désormais une ventilation par utilisateur de l'activité Copilot CLI : indicateur d'usage (used_cli), nombre de sessions et requêtes, consommation de tokens, et version CLI par utilisateur. Un complément précieux pour le suivi d'adoption, la planification des déploiements et l'allocation de coûts.

→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

5. Gouvernance MCP en entreprise dans Visual Studio — Mise à jour mars 2026

La mise à jour de mars de Visual Studio introduit la gouvernance des serveurs MCP au niveau entreprise : les administrateurs peuvent désormais définir une liste blanche des serveurs MCP autorisés dans leurs organisations via les politiques Copilot. Protège les données sensibles et assure la conformité pour les équipes utilisant des agents avec des extensions MCP.

→ Changelog: GitHub Copilot in Visual Studio — March update
