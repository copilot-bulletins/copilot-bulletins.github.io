GitHub Copilot Bulletin — Biweekly #1

Du 2 au 16 mars 2026 · Admins Edition

🎯 Le fil de cette édition

Ces deux dernières semaines, GitHub a renforcé le contrôle administrateur sur les capacités agentiques de Copilot. Revue de code agentique en GA, nouvelles options de gouvernance pour le coding agent, métriques d'utilisation étendues au niveau utilisateur et routage réseau par abonnement : les administrateurs disposent désormais d'outils plus fins pour encadrer le déploiement de Copilot à l'échelle.

⚠️ Attention

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline : 26 mars 2026 (Gemini 3 Pro) et 1er avril 2026 (GPT-5.1, GPT-5.1-Codex, GPT-5.1-Codex-Mini, GPT-5.1-Codex-Max) — Vérifiez vos politiques de modèles et activez l'accès aux modèles alternatifs avant ces dates.

🚀 Passez à l'action

  • Configurez la revue de code automatique par Copilot sur vos dépôts ou à l'échelle de votre organisation.
  • Vérifiez les paramètres du coding agent pour contrôler l'approbation automatique des workflows Actions.
  • Mettez à jour vos règles réseau avec les nouveaux endpoints de routage réseau par abonnement.
  • Activez les modèles alternatifs dans vos politiques Copilot avant la dépréciation de Gemini 3 Pro et GPT-5.1.
  • Des questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 5 des mises à jour

1. La revue de code Copilot passe à l'architecture agentique

Copilot code review est désormais en GA avec une architecture agentique qui explore le contexte du dépôt pour des retours plus précis et moins de bruit. Les organisations sur self-hosted runners doivent effectuer une configuration initiale.

→ Changelog: Copilot code review now runs on an agentic architecture

2. Approbation optionnelle des workflows Actions pour le coding agent

Un nouveau paramètre de dépôt permet aux administrateurs de désactiver l'approbation humaine des workflows Actions déclenchés par le coding agent. Par défaut, l'approbation reste requise pour préserver la sécurité.

→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

3. Métriques d'utilisation CLI au niveau utilisateur

Les métriques Copilot incluent désormais l'activité CLI par utilisateur : nombre de requêtes, sessions et consommation de tokens. Un outil essentiel pour identifier les besoins d'accompagnement et planifier le déploiement.

→ Changelog: Copilot usage metrics now includes user-level GitHub Copilot CLI activity

4. Routage réseau par abonnement du coding agent en vigueur

Les endpoints réseau du coding agent sont désormais différenciés par plan (Business, Enterprise, Pro). Les équipes utilisant des self-hosted runners ou Azure private networking doivent mettre à jour leurs règles de pare-feu.

→ Changelog: Network configuration changes for Copilot coding agent now in effect

5. Copilot Memory activée par défaut pour Pro et Pro+

Copilot Memory, qui permet à Copilot de retenir le contexte d'un dépôt entre les sessions, est désormais activée par défaut. Les administrateurs enterprise et organisation conservent le contrôle total via les politiques Copilot.

→ Changelog: Copilot Memory now on by default for Pro and Pro+ users in public preview
