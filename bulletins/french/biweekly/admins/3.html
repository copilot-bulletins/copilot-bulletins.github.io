Biweekly #3 · Du 1er avril au 15 avril 2026 · Admins Edition

🎯 Le fil de cette édition

Cette quinzaine marque une étape majeure dans la gouvernance de Copilot à l'échelle de l'entreprise : la résidence des données US/EU et la conformité FedRAMP passent en disponibilité générale. Simultanément, GitHub enrichit les métriques d'adoption — cloud agent, revue de code, CLI — offrant aux administrateurs une visibilité complète et unifiée de l'utilisation de Copilot dans leur organisation.

⚠️ Attention

  • Enforcing new limits and retiring Opus 4.6 Fast from Copilot Pro+ — Retrait immédiat — Le modèle Opus 4.6 Fast est retiré des abonnements Copilot Pro+. Les utilisateurs affectés doivent basculer vers Opus 4.6 comme alternative ; de nouveaux plafonds d'utilisation sont également mis en place progressivement sur tous les plans.
  • Pausing new GitHub Copilot Pro trials — Action requise — Les nouveaux essais Copilot Pro sont suspendus jusqu'à nouvel ordre en raison d'abus détectés. Orientez les nouveaux utilisateurs vers Copilot Free ou un abonnement payant en attendant le rétablissement des essais.

🚀 Passez à l'action

  • Activez la résidence des données (US/EU) ou la conformité FedRAMP depuis les paramètres Copilot de votre entreprise.
  • Exploitez l'API des métriques d'utilisation Copilot pour suivre l'adoption du cloud agent et mesurer l'impact des revues de code automatisées sur vos cycles de développement.
  • Activez les politiques d'accès pour les agents tiers (Claude, Codex) depuis la page Politiques et fonctionnalités de votre entreprise.
  • Vérifiez vos tableaux de bord de métriques : les totaux de haut niveau incluent désormais l'activité CLI. Ajustez vos seuils si nécessaire.
  • Des questions ou besoin d'aide ? Contactez votre équipe GitHub, nous sommes là pour vous accompagner.

🏆 Top 5 des mises à jour

1. Résidence des données Copilot US/EU et conformité FedRAMP disponibles en GA

GitHub Copilot prend désormais en charge la résidence des données pour les régions US et EU, garantissant que tous les traitements d'inférence restent dans la géographie désignée. Les clients gouvernementaux américains bénéficient en outre de la conformité FedRAMP Moderate. Les administrateurs d'entreprise activent ces restrictions depuis leurs paramètres Copilot, avec une majoration de 10 % du multiplicateur de requêtes premium.

→ Changelog: Copilot data residency in US + EU and FedRAMP compliance now available

2. Les métriques d'utilisation agrègent désormais les utilisateurs actifs du cloud agent

Les rapports au niveau entreprise et organisation incluent maintenant des comptages d'utilisateurs actifs du Copilot cloud agent : quotidien, hebdomadaire et mensuel. Ces champs permettent de suivre l'adoption de l'agent sans agréger manuellement les données par utilisateur, et s'ajoutent aux métriques existantes pour une vue complète de l'usage de Copilot.

→ Changelog: Copilot usage metrics now aggregate Copilot cloud agent active user counts

3. Métriques de fusion pour les PR révisées par Copilot dans l'API

Deux nouveaux champs rejoignent l'API des métriques d'utilisation : nombre de PR fusionnées après revue Copilot et délai médian de fusion pour ces PR. Combinées aux métriques sur les PR créées par le cloud agent, elles offrent une vue bout en bout de la contribution de Copilot dans le cycle de vie des pull requests.

→ Changelog: Copilot-reviewed pull request merge metrics now in the usage metrics API

4. L'activité Copilot CLI intégrée dans les totaux des métriques d'utilisation

L'activité Copilot CLI est désormais incluse dans les totaux de haut niveau de l'API et apparaît dans les ventilations par fonctionnalité (feature=copilot_cli). Les administrateurs disposent d'une vue unifiée IDE + CLI sans agrégation manuelle. À noter : les totaux de haut niveau reflètent maintenant IDE + CLI — vérifiez vos tableaux de bord si ces valeurs y sont utilisées.

→ Changelog: Copilot CLI activity now included in usage metrics totals and feature breakdowns

5. Sélection de modèle pour les agents tiers Claude et Codex sur github.com

Les agents de codage tiers Claude et Codex supportent désormais la sélection de modèle sur github.com, comme le cloud agent Copilot. Pour les abonnements Business et Enterprise, les administrateurs doivent activer les politiques Anthropic Claude ou OpenAI Codex depuis les paramètres d'entreprise pour en autoriser l'accès.

→ Changelog: Model selection for Claude and Codex agents on github.com
