🎯 Le fil de cette édition

Ce mois-ci, GitHub Copilot franchit un cap stratégique : l'IA ne se contente plus d'assister les développeurs, elle s'intègre comme infrastructure d'exécution à l'échelle de l'entreprise. Du SDK programmable aux 60 millions de revues de code automatisées, la plateforme démontre un passage de l'expérimentation à l'industrialisation, avec les outils de mesure d'impact pour le prouver.

🏆 Top 5 des mises à jour du mois

1. Copilot code review passe à l'architecture agentique — 60 millions de revues de code

Copilot code review fonctionne désormais sur une architecture agentique en disponibilité générale. Le système explore le contexte du dépôt pour produire des retours plus précis et moins de bruit. Avec 60 millions de revues réalisées, plus de 12 000 organisations l'utilisent automatiquement sur chaque pull request, et WEX rapporte environ 30 % de code livré en plus.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Blog: 60 million Copilot code reviews and counting

2. GitHub Copilot SDK en preview publique — l'exécution agentique devient programmable

Le Copilot SDK permet d'intégrer le moteur d'exécution agentique de Copilot directement dans vos applications métier. Au lieu de construire votre propre couche d'orchestration, vos systèmes peuvent désormais déléguer des tâches multi-étapes, accéder à des outils via MCP, et exécuter des workflows adaptatifs dans vos propres produits.

→ Blog: The era of "AI as text" is over. Execution is the new interface.

3. GPT-5.4 disponible dans GitHub Copilot

GPT-5.4, le dernier modèle agentique d'OpenAI, est en disponibilité générale dans Copilot. Disponible sur tous les IDE (VS Code, JetBrains, Visual Studio, Xcode, Eclipse), GitHub.com et le Copilot CLI, il offre un raisonnement logique amélioré pour les processus complexes et multi-étapes. Les administrateurs doivent activer la politique GPT-5.4 dans les paramètres Copilot.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. Copilot metrics en disponibilité générale — mesurez l'adoption et l'impact

La suite complète de métriques Copilot est désormais en GA : tableaux de bord d'utilisation et de génération de code, API enterprise/organisation/utilisateur, télémétrie CLI et mode plan. Les Enterprise AI Controls et le plan de contrôle des agents sont également en GA, offrant une supervision centralisée de l'activité agentique avec de nouveaux filtres par statut, dépôt et utilisateur.

Sources:
→ Changelog: Copilot metrics is now generally available
→ Changelog: Enterprise AI Controls & agent control plane now generally available
→ Changelog: Discover and manage agent activity with new session filters

5. Copilot coding agent s'intègre à Jira et accélère l'autonomie

Le coding agent Copilot peut désormais être assigné directement depuis Jira : il analyse le ticket, implémente les changements et ouvre une pull request automatiquement. En parallèle, un nouveau paramètre permet aux administrateurs de supprimer l'approbation manuelle des workflows Actions pour les PR du coding agent, accélérant la boucle de feedback.

Sources:
→ Changelog: GitHub Copilot coding agent for Jira is now in public preview
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
