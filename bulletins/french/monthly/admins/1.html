Monthly #1 · Mars 2026 · Admins Edition

🎯 Le fil de cette édition

Ce mois-ci, GitHub pose les fondations d'une gouvernance IA mature pour les entreprises. Avec la disponibilité générale de l'AI Controls, des métriques d'utilisation et de la code review agentique, les administrateurs disposent désormais d'outils complets pour superviser, mesurer et piloter l'adoption de Copilot à l'échelle de leur organisation.

⚠️ Attention

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline: 26 mars (Gemini 3 Pro), 1er avril (GPT-5.1) — Vérifiez vos politiques de modèles et activez les alternatives recommandées (Gemini 3.1 Pro, GPT-5.3-Codex) avant ces dates.

🚀 Passez à l'action

  • Configurez les AI Controls et l'agent control plane dans les paramètres de votre enterprise pour centraliser la gouvernance des agents.
  • Activez Claude et Codex pour vos utilisateurs Business dans les paramètres AI Controls de l'enterprise, puis au niveau de l'organisation.
  • Explorez le nouveau dashboard de métriques Copilot pour suivre l'adoption et l'utilisation de Copilot dans votre entreprise.
  • Désignez des AI managers pour décentraliser l'administration IA avec des rôles personnalisés enterprise.
  • Questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 5 des mises à jour du mois

1. AI Controls et agent control plane en disponibilité générale

Les administrateurs enterprise disposent désormais d'un panneau de gouvernance IA centralisé : rôles personnalisés, supervision des sessions d'agents, journal d'audit dédié, politiques d'agents personnalisés et registre MCP, le tout depuis un point d'accès unique.

→ Changelog: Enterprise AI Controls & agent control plane now generally available

2. Métriques Copilot en GA avec couverture élargie

Les métriques d'utilisation Copilot sont maintenant en GA avec dashboards, API et données granulaires. La couverture inclut désormais la télémétrie CLI, le mode plan, les noms d'utilisateurs EMU cohérents et de nouveaux filtres de sessions d'agents.

Sources:
→ Changelog: Copilot metrics is now generally available
→ Changelog: Copilot usage metrics now includes enterprise-level GitHub Copilot CLI activity
→ Changelog: Copilot metrics now includes plan mode
→ Changelog: Discover and manage agent activity with new session filters

3. Claude et Codex disponibles pour Business et Pro, GPT-5.4 en GA

Les agents Claude (Anthropic) et Codex (OpenAI) sont désormais accessibles aux utilisateurs Copilot Business et Pro, inclus dans l'abonnement existant. GPT-5.4 est aussi en GA. Les administrateurs doivent activer ces modèles via les politiques enterprise et organisation.

Sources:
→ Changelog: Claude and Codex now available for Copilot Business & Pro users
→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. Code review Copilot sur architecture agentique en GA

La code review Copilot utilise désormais une architecture agentique pour analyser le contexte complet du dépôt. Résultat : des commentaires plus pertinents et moins de bruit. Les équipes utilisant des runners self-hosted doivent effectuer une configuration initiale.

→ Changelog: Copilot code review now runs on an agentic architecture

5. Nouveaux leviers d'administration : Content Exclusion API, approbation workflows et réseau

L'API REST Content Exclusion permet de gérer les règles d'exclusion de contenu par programme. Les administrateurs de dépôt peuvent désactiver l'approbation manuelle des workflows Actions pour le coding agent. Les changements de configuration réseau (routage par abonnement) sont désormais en vigueur.

Sources:
→ Changelog: Copilot Content Exclusion REST API in public preview
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
→ Changelog: Network configuration changes for Copilot coding agent now in effect
