Monthly #2 · Avril 2026 · Devs Edition

🎯 Le fil de cette édition

En avril 2026, l'agentique autonome franchit un cap décisif pour les développeurs. Copilot cloud agent s'affranchit des workflows de pull request pour explorer, planifier et itérer sur des branches à votre rythme. Dans l'IDE et le CLI, de nouveaux primitives — SDK public, /fleet, agents personnalisés — concrétisent une collaboration homme-machine d'un nouveau genre.

⚠️ Attention

  • GPT-5.1-Codex, GPT-5.1-Codex-Max et GPT-5.1-Codex-Mini dépréciés — Action requise — Ces trois modèles ont été retirés le 1er avril 2026. Migrez vers GPT-5.3-Codex et vérifiez les politiques de modèles dans vos paramètres Copilot.
  • Dépréciation à venir de Claude Sonnet 4 dans GitHub Copilot — Échéance : 1er mai 2026 — Migrez vers Claude Sonnet 4.6 avant cette date. Vérifiez que le modèle alternatif est activé dans vos politiques de modèles.
  • Mise à jour de la déclaration de confidentialité et des CGU — Échéance : 24 avril 2026 — À partir du 24 avril, GitHub utilisera les données d'interaction des utilisateurs Copilot Free, Pro et Pro+ pour entraîner ses modèles IA. Désactivez cette option dans vos paramètres Copilot si vous ne souhaitez pas y participer. Copilot Business et Enterprise ne sont pas concernés.

🚀 Passez à l'action

  • Testez la recherche en profondeur et la planification avec Copilot cloud agent depuis le panneau Agents de votre dépôt.
  • Mentionnez @copilot dans vos pull requests pour déléguer corrections de tests, résolution de conflits ou toute autre modification — consultez la documentation dédiée.
  • Explorez le Copilot SDK (public preview) pour intégrer des capacités agentiques dans vos propres outils, disponible en Node.js/TypeScript, Python, Go, .NET et Java.
  • Essayez /fleet dans Copilot CLI pour exécuter vos tâches multi-fichiers en parallèle — consultez la documentation /fleet pour bien démarrer.
  • Questions ou besoin d'accompagnement ? Contactez votre équipe GitHub, nous sommes là pour vous aider.

🏆 Top 5 des mises à jour du mois

1. Copilot cloud agent : recherche, planification et itération sans PR automatique

Copilot cloud agent peut désormais explorer votre dépôt, proposer un plan d'implémentation avant d'écrire du code, et travailler sur une branche sans ouvrir de PR automatiquement. Lancez la pull request uniquement quand vous êtes satisfait du diff. Disponible depuis le panneau Agents sur GitHub.com.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. @copilot directement dans vos pull requests

Mentionnez @copilot dans un commentaire de pull request pour déléguer corrections de tests, résolution de conflits de fusion ou toute autre modification. L'agent travaille dans son environnement cloud dédié, valide les tests et pousse directement sur votre branche avant de demander une revue.

Sources : → Changelog: Ask @copilot to make changes to a pull request
→ Changelog: Ask @copilot to resolve merge conflicts on pull requests

3. Copilot SDK en préversion publique

Le SDK GitHub Copilot expose le même runtime agentique que Copilot cloud agent et Copilot CLI, disponible en Node.js/TypeScript, Python, Go, .NET et Java. Outils personnalisés, streaming, sessions multi-tours, support OpenTelemetry et BYOK inclus pour intégrer des capacités IA dans vos propres applications.

→ Changelog: Copilot SDK in public preview

4. Visual Studio mars 2026 : agents personnalisés, skills et outils de profilage

Visual Studio 2026 (mars) permet de définir des agents personnalisés via des fichiers .agent.md, des agent skills réutilisables, et un outil find_symbol pour la navigation de code. Le nouveau Profiling Agent analyse automatiquement les performances à l'exécution et pendant le débogage avec les PerfTips.

→ Changelog: GitHub Copilot in Visual Studio — March update

5. /fleet dans Copilot CLI : agents en parallèle

La commande /fleet décompose une tâche complexe en sous-tâches indépendantes exécutées simultanément par plusieurs agents. Refactorisations multi-fichiers, documentation ou features transverses : chaque sous-agent travaille dans son périmètre pendant que l'orchestrateur coordonne et synthétise le résultat final.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI
