Monthly (Long) #2 · Abril 2026 · Admins Edition

🎯 El hilo de esta edición

Marzo y principios de abril marcaron un punto de inflexión en la madurez de gobernanza de GitHub Copilot para los equipos de administración. Por primera vez, los administradores de organización pueden controlar de forma centralizada el entorno de ejecución del agente en la nube: el tipo de runner, el firewall y las instrucciones personalizadas ya se gestionan a nivel de organización, no repositorio a repositorio. En paralelo, el agente amplía su alcance más allá de los pull requests y ahora firma sus commits, abriendo la puerta a repositorios con políticas de commit firmado obligatorio.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Fecha de depreciación: 1 de abril de 2026 — Verifique las políticas de modelos de su organización y habilite GPT-5.3-Codex como alternativa desde la configuración de modelos de Copilot. No se requiere ninguna acción para eliminar los modelos deprecados.

🚀 Pasa a la acción

  • Configure los runners predeterminados para Copilot cloud agent a nivel de organización desde Settings → Copilot → Cloud agent y bloquee la configuración para evitar que los repositorios la anulen.
  • Establezca la política del firewall del agente a nivel de organización para proteger contra la exfiltración de datos y la inyección de prompts desde Customizing the agent firewall.
  • Active las instrucciones personalizadas de organización para guiar el comportamiento de Copilot en chat, revisiones de código y el agente en la nube: vaya a Settings → Copilot → Custom instructions.
  • Revise las métricas de uso de CLI de Copilot por usuario en los reportes de organización para identificar patrones de adopción y planificar la expansión.
  • ¿Preguntas o necesita asistencia? Contacte a su equipo de GitHub, estamos aquí para ayudarle.

🏆 Top 7 de actualizaciones del mes

1. Controles de organización para Copilot cloud agent: runners y firewall

Los administradores de organización pueden ahora establecer el runner predeterminado del agente en la nube a nivel de organización y bloquearlo para que los repositorios no puedan modificarlo. Igualmente, el firewall del agente se gestiona centralmente: activación, listas de dominios permitidos y control sobre la personalización por repositorio.

Sources:
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

2. Instrucciones personalizadas de organización: disponibilidad general

Las instrucciones personalizadas de organización para Copilot alcanzan disponibilidad general para Copilot Business y Copilot Enterprise. Los administradores definen instrucciones que se aplican automáticamente en Copilot Chat en GitHub.com, revisiones de código y el agente en la nube, sin configuración individual por repositorio.

→ Changelog: Copilot organization custom instructions are generally available

3. Copilot cloud agent firma sus commits

El agente ahora firma cada commit como "Verified" en GitHub, garantizando su autenticidad. Esto permite utilizarlo en repositorios con la regla de protección "Require signed commits" habilitada — una restricción que anteriormente bloqueaba completamente el uso del agente.

→ Changelog: Copilot cloud agent signs its commits

4. Métricas de uso del CLI de Copilot por usuario en reportes de organización

Los administradores de organización disponen ahora de desgloses por usuario para la actividad del CLI: estado de uso, recuentos de sesiones y solicitudes, consumo de tokens y versión de CLI por usuario. Completa la cobertura total de métricas de CLI a nivel de empresa, usuario y organización.

→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

5. Gobernanza de MCP y agentes personalizados en Visual Studio

La actualización de marzo de Visual Studio incorpora la gestión centralizada de servidores MCP mediante listas de permisos organizacionales, manteniendo el control sobre el acceso a datos sensibles. Los equipos también pueden definir agentes personalizados como archivos .agent.md en el repositorio, disponibles en toda la organización.

→ Changelog: GitHub Copilot in Visual Studio — March update

6. Copilot cloud agent más allá de los pull requests

El agente en la nube ya no está limitado al flujo de pull requests. Ahora puede trabajar en una rama sin crear un PR, generar planes de implementación para revisión antes de escribir código, y realizar investigación profunda en el repositorio. Los administradores deben conocer estos nuevos modos al revisar sus políticas de uso del agente.

→ Changelog: Research, plan, and code with Copilot cloud agent

7. SDK de Copilot en vista previa pública

El SDK de Copilot está disponible en vista previa pública para 5 lenguajes (Node.js, Python, Go, .NET, Java), exponiendo el mismo runtime del agente que impulsa el agente en la nube y el CLI. Permite a equipos de plataforma integrar capacidades agénticas en aplicaciones internas, con soporte BYOK (traiga su propia clave API).

→ Changelog: Copilot SDK in public preview
