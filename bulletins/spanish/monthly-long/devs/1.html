Monthly (Long) #1 · Marzo 2026 · Devs Edition

🎯 El hilo de esta edición

Marzo marca un punto de inflexión para los desarrolladores que usan GitHub Copilot: la era de los agentes autónomos es oficial. Con el coding agent en disponibilidad general, una nueva arquitectura agéntica en code review, y VS Code y JetBrains recibiendo mejoras masivas en capacidades de agentes, el flujo de trabajo del desarrollador se transforma. A esto se suma GPT-5.4 como nuevo modelo de referencia y Copilot Memory activada por defecto, consolidando un ecosistema donde los agentes no solo ejecutan tareas, sino que aprenden y se adaptan al contexto de cada repositorio.

⚠️ Atención

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline: 26 de marzo (Gemini 3 Pro) y 1 de abril (GPT-5.1, GPT-5.1-Codex, GPT-5.1-Codex-Mini, GPT-5.1-Codex-Max) — Migra tus flujos de trabajo a los modelos alternativos sugeridos. Consulta los modelos disponibles para planificar la transición.

🚀 Pasa a la acción

  • Prueba GPT-5.4 seleccionándolo en el model picker de tu IDE o desde los modelos disponibles en Copilot.
  • Delega tu primera tarea al coding agent directamente desde un issue o el panel de agentes. Consulta la documentación del coding agent.
  • Configura revisiones automáticas de Copilot en cada pull request de tu repositorio u organización con la guía de configuración.
  • Activa Copilot Memory en tu repositorio para que los agentes aprendan y mejoren con cada interacción. Más detalles en la documentación de Copilot Memory.
  • ¿Preguntas o necesitas ayuda? Contacta a tu equipo de GitHub, estamos para apoyarte.

🏆 Top 8 de actualizaciones del mes

1. Copilot coding agent en disponibilidad general con integración Jira y más control

El agente autónomo de Copilot está disponible para todos los suscriptores de pago. Delega tareas desde issues, VS Code o el panel de agentes, y Copilot abre un draft PR trabajando en segundo plano. Además, ahora se integra con Jira, permite elegir modelo al mencionar @copilot en PRs, adjuntar imágenes a sesiones, y opcionalmente omitir aprobación de Actions workflows.

Sources:
→ Changelog: Copilot coding agent is now generally available
→ Changelog: GitHub Copilot coding agent for Jira is now in public preview
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
→ Changelog: Pick a model for @copilot in pull request comments
→ Changelog: Add images to agent sessions

2. VS Code v1.110: hooks, plugins de agentes y gestión avanzada de contexto

La release de febrero de VS Code transforma la experiencia con agentes. Hooks permiten ejecutar código en eventos clave del agente, los agent plugins empaquetan skills y herramientas, y /autoApprove activa auto-aprobación desde el chat. Además, context compaction mantiene el historial relevante, el subagente Explore delega investigación a modelos ligeros, y Copilot CLI se integra nativamente en VS Code.

→ Changelog: GitHub Copilot in Visual Studio Code v1.110 – February release

3. GPT-5.4 disponible en GitHub Copilot

GPT-5.4 de OpenAI llega a todos los planes de pago con mejoras significativas en razonamiento lógico y ejecución de tareas multi-paso. Disponible en VS Code, JetBrains, Visual Studio, Xcode, Eclipse, github.com, GitHub Mobile y GitHub CLI. Los administradores deben habilitar la política del modelo.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. JetBrains IDEs: agentes personalizados, sub-agentes y plan agent en GA

Importante actualización para JetBrains con agentes personalizados, sub-agentes y plan agent ahora en GA. Se añade soporte para agent hooks en preview, auto-aprobación MCP, archivos de instrucciones AGENTS.md y CLAUDE.md, panel de razonamiento para modelos extendidos, e indicador de uso de ventana de contexto.

→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs

5. Code review agéntico: arquitectura renovada y revisión desde la CLI

Copilot code review adopta una arquitectura agéntica que explora el repositorio para entender la lógica y arquitectura del código. Genera comentarios de mayor señal con menos ruido. Además, ahora puedes solicitar revisión de Copilot directamente desde la terminal con gh pr edit --add-reviewer @copilot`` o al crear un PR.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Changelog: Request Copilot code review from GitHub CLI
→ Blog: 60 million Copilot code reviews and counting

6. Figma MCP server: flujo bidireccional diseño-código desde VS Code

Los usuarios de Copilot pueden ahora conectar el servidor MCP de Figma para traer contexto de diseño al código y enviar UI renderizada de vuelta a Figma como frames editables. Esto habilita un flujo iterativo completo: genera código desde un diseño, envía tu UI a Figma, y trae las actualizaciones de vuelta.

→ Changelog: Figma MCP server can now generate design layers from VS Code

7. Copilot Memory activada por defecto para Pro y Pro+

Copilot Memory se habilita automáticamente para usuarios Pro y Pro+. Permite que Copilot retenga conocimiento a nivel de repositorio (convenciones, patrones arquitectónicos, dependencias) y lo aplique en coding agent, code review y CLI. Las memorias se comparten entre agentes y expiran automáticamente a los 28 días.

→ Changelog: Copilot Memory now on by default for Pro and Pro+ users in public preview

8. Explora repositorios con Copilot en la web

Copilot en github.com permite ahora explorar la estructura de archivos de un repositorio mientras chateas. Navega el árbol de archivos, visualiza contenido y añade referencias automáticamente al contexto de tu conversación. Ideal para entender la estructura de proyectos nuevos rápidamente.

→ Changelog: Explore a repository using Copilot on the web
