Monthly (Long) #2 · Marzo 2026 · Devs Edition

🎯 El hilo de esta edición

Marzo 2026 marcó un punto de inflexión en el desarrollo asistido por agentes. GitHub lanzó el Copilot SDK en preview pública, habilitando que cualquier desarrollador embeba las capacidades agénticas de Copilot en sus propias aplicaciones y flujos de trabajo.

Al mismo tiempo, herramientas como /fleet en Copilot CLI y las mejoras del cloud agent transforman el ciclo de desarrollo cotidiano: más paralelismo, más autonomía y mayor transparencia. Visual Studio suma agentes personalizados y skills reutilizables. La tendencia es inequívoca: el desarrollo ya no es secuencial ni solitario, sino orquestado y colaborativo.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Deprecación: 1 de abril de 2026 — Migra a GPT-5.3-Codex. Revisa los modelos soportados para actualizar tus flujos de trabajo e integraciones.

🚀 Pasa a la acción

  • Explora el Copilot SDK para integrar las capacidades agénticas de Copilot en tus propias aplicaciones (disponible en Node.js, Python, Go, .NET y Java).
  • Instala o actualiza GitHub Copilot CLI y prueba el comando /fleet con tu próxima tarea compleja para ejecutar subtareas en paralelo.
  • Configura agent skills en tu repositorio para enseñar a Copilot cómo realizar tareas especializadas de forma repetible.
  • Crea agentes personalizados con archivos .agent.md para especializar a Copilot según tus convenciones y flujos de trabajo del equipo.
  • ¿Preguntas o necesitas ayuda? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 7 de actualizaciones del mes

1. SDK de Copilot en preview pública

El Copilot SDK está disponible en preview pública en cinco lenguajes: Node.js, Python, Go, .NET y Java. Integra el runtime agéntico de Copilot en tus propias aplicaciones con herramientas personalizadas, streaming de respuestas, adjuntos de imágenes y soporte OpenTelemetry incluidos.

→ Changelog: Copilot SDK in public preview

2. /fleet: ejecución de múltiples agentes en paralelo en Copilot CLI

El nuevo comando /fleet en Copilot CLI descompone tareas complejas en subtareas independientes y las ejecuta en paralelo mediante subagentes. Ideal para refactorizaciones cruzadas, documentación simultánea o implementaciones que abarcan API, UI y tests. Cada subagente mantiene su propio contexto.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

3. GitHub Copilot en Visual Studio: agentes personalizados, skills y nuevas herramientas

La actualización de marzo de Visual Studio 2026 trae agentes personalizados en archivos .agent.md, skills reutilizables autodescubribles, la herramienta find_symbol con navegación semántica (C++, C#, TypeScript…) y análisis de rendimiento desde Copilot directamente en el depurador.

→ Changelog: GitHub Copilot in Visual Studio — March update

4. Cloud agent: commits firmados y controles a nivel organización

El cloud agent ahora firma todos sus commits ("Verified" en GitHub), compatible con la regla "Require signed commits". Además, los admins de organización pueden definir el runner predeterminado y la política de firewall a escala, simplificando el despliegue del agente en toda la organización.

Sources:
→ Changelog: Copilot cloud agent signs its commits
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

5. Desarrollo impulsado por agentes: lecciones del equipo de Applied Science

El equipo de Copilot Applied Science entregó 11 agentes y 4 skills en menos de 3 días (+28.858 líneas en 345 archivos). Claves para replicarlo: prompts conversacionales, arquitectura orientada a agentes y la filosofía "blame process, not agents".

→ Blog: Agent-driven development in Copilot Applied Science

6. Métricas de uso de Copilot CLI: cobertura completa en todos los niveles

GitHub completó la cobertura de métricas de Copilot CLI, ahora disponibles a nivel empresa, organización y usuario individual en informes de 1 y 28 días. Datos por usuario: sesiones, solicitudes, tokens y versión del CLI instalada, útil para planificar rollouts.

→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

7. Squad: orquestación multi-agente nativa del repositorio

Squad, proyecto open source sobre Copilot, inicializa en dos comandos un equipo de agentes (coordinador, frontend, backend, tester) en tu repositorio. Usa el patrón "drop-box": decisiones en archivos Markdown versionados como memoria compartida, sin infraestructura compleja.

→ Blog: How Squad runs coordinated AI agents inside your repository
