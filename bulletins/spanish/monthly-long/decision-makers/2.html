Monthly (Long) #2 · Abril 2026 · Decision-Makers Edition

🎯 El hilo de esta edición

Este mes marca un punto de inflexión en la evolución de GitHub Copilot como plataforma empresarial: el agente de nube amplía su alcance más allá de los pull requests hacia ciclos completos de investigación, planificación e implementación; el lanzamiento del SDK abre el motor de IA de GitHub a aplicaciones y flujos de trabajo propios; y un conjunto de controles organizacionales sin precedente permite desplegar el agente con las garantías que exige un entorno de producción. La pregunta ya no es si adoptar IA generativa, sino con qué velocidad escalar de forma segura.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Ya retirados: 1 de abril de 2026 — Actualiza workflows e integraciones al modelo GPT-5.3-Codex; los administradores de Copilot Enterprise deben habilitar el nuevo modelo en sus políticas de modelos.
  • Próxima deprecación de Claude Sonnet 4 — Fecha límite: 1 de mayo de 2026 — Migra a Claude Sonnet 4.6 antes de esa fecha; verifica que las políticas de tu organización permitan el acceso al modelo alternativo en la configuración de modelos de Copilot.

🚀 Pasa a la acción

  • Habilita Copilot cloud agent en tu organización y configura los controles de runner y firewall agéntico a nivel organizacional.
  • Define instrucciones personalizadas de organización para alinear el comportamiento de Copilot con los estándares de tu equipo.
  • Explora el SDK de Copilot para integrar las capacidades agénticas en tus propias plataformas internas.
  • Revisa las métricas de uso de Copilot para cuantificar la adopción del agente de nube y del CLI por usuario.
  • ¿Preguntas o necesitas apoyo? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 7 de actualizaciones del mes

1. Copilot cloud agent: investigar, planificar y construir en un solo flujo

Copilot cloud agent (antes «Copilot coding agent») ya no se limita a abrir pull requests: ahora puede investigar un repositorio en profundidad, generar un plan de implementación para revisión del equipo antes de escribir código, y trabajar en una rama sin crear un PR hasta que el equipo esté listo. Mayor control editorial, menor fricción operativa.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. Gobernanza empresarial del agente de nube: controles a escala

Cuatro actualizaciones que hacen del agente de nube una opción viable en producción: runner por defecto configurable y bloqueeable a nivel de organización, gestión del firewall agéntico centralizada, instrucciones personalizadas de organización ahora en disponibilidad general, y commits firmados (Verified) que desbloquean el agente en repositorios con branch protection estricta.

Sources:
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent
→ Changelog: Copilot cloud agent signs its commits
→ Changelog: Copilot organization custom instructions are generally available

3. SDK de Copilot en vista previa pública

GitHub abre el motor de ejecución agéntico que impulsa Copilot cloud agent y Copilot CLI para que cualquier organización pueda integrarlo en sus propias aplicaciones y plataformas internas. Disponible en cinco lenguajes (Node.js, Python, Go, .NET, Java), con soporte para BYOK, streaming, OpenTelemetry y un framework de permisos granular.

→ Changelog: Copilot SDK in public preview

4. Copilot donde trabajan tus equipos: Jira, Slack y móvil

La integración con Jira suma selección de modelo y acceso a Confluence vía MCP. En Slack, @GitHub permite crear issues estructurados desde lenguaje natural. GitHub Mobile estrena pestaña Copilot en la barra de navegación y registros de sesión nativos. Copilot llega al contexto donde los equipos ya trabajan, sin cambiar flujos existentes.

Sources:
→ Changelog: GitHub Copilot for Jira — Public preview enhancements
→ Changelog: Create issues from Slack with Copilot
→ Changelog: GitHub Mobile: Stay in flow with a refreshed Copilot tab and native session logs

5. Visibilidad completa de la adopción: métricas agénticas y CLI por usuario

Las métricas de Copilot ahora reflejan toda la actividad agéntica: el campo used_copilot_coding_agent identifica qué usuarios usan el agente de nube, y los informes de organización incluyen el desglose de actividad CLI por usuario con conteo de sesiones, consumo de tokens y versión instalada. Una base de datos concreta para cuantificar ROI y planificar expansiones de licencias.

Sources:
→ Changelog: Copilot usage metrics now identify active Copilot coding agent users
→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

6. GitHub Copilot en Visual Studio — actualización de marzo: agentes personalizados y gobernanza MCP

La actualización de marzo de Visual Studio 2026 introduce agentes Copilot definibles como archivos .agent.md en el repositorio, gobernanza empresarial de servidores MCP mediante políticas de allowlist, skills de agente reutilizables y análisis de rendimiento con Copilot Profiler. Para organizaciones con desarrollo .NET, C++ y C#, una extensión significativa del perímetro agéntico de Copilot.

→ Changelog: GitHub Copilot in Visual Studio — March update

7. Actualización de política de privacidad y uso de datos de Copilot

A partir del 24 de abril, GitHub usará datos de interacción de Copilot Free, Pro y Pro+ para mejorar sus modelos de IA, salvo que los usuarios desactiven la opción en sus ajustes. Las cuentas Copilot Business y Copilot Enterprise no se ven afectadas: sus DPA permanecen vigentes y sus datos de interacción no se utilizan para entrenamiento sin consentimiento explícito.

→ Changelog: Updates to our Privacy Statement and Terms of Service: How we use your data
