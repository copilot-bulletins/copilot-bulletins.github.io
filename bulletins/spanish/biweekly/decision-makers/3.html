Biweekly #3 · Del 1 al 15 de abril de 2026 · Decision-Makers Edition

🎯 El hilo de esta edición

Esta quincena, GitHub Copilot consolidó su posición como plataforma empresarial. Las actualizaciones muestran una estrategia triple: garantizar conformidad regulatoria (EE. UU., UE y FedRAMP), extender las capacidades de IA agéntica a aplicaciones propias mediante el nuevo SDK, y reforzar los controles de gobernanza a escala organizativa. El mensaje es claro: Copilot está listo para los entornos más exigentes.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Action required — Estos modelos quedaron retirados el 1 de abril de 2026 en todas las experiencias de Copilot. Migre sus flujos de trabajo e integraciones a GPT-5.3-Codex. Los administradores de Copilot Enterprise deben verificar y habilitar el acceso a modelos alternativos desde la configuración de políticas.

🚀 Pasa a la acción

  • Active la residencia de datos y/o FedRAMP para su empresa desde la configuración de residencia de datos de Copilot (las políticas están desactivadas por defecto; es necesario habilitarlas explícitamente).
  • Explore el Copilot SDK e integre capacidades del agente en sus propias aplicaciones con la guía de inicio oficial.
  • Configure controles de runner a nivel organización para el agente en la nube desde la documentación para administradores.
  • Consulte la API de métricas de uso para obtener datos sobre la adopción del agente en la nube desde la documentación de la API.
  • ¿Preguntas o necesita apoyo? Contacte a su equipo de GitHub, estamos aquí para ayudar.

🏆 Top 5 de actualizaciones

1. Residencia de datos en EE. UU. + UE y conformidad FedRAMP para Copilot

GitHub Copilot garantiza ahora que todo el procesamiento de inferencia y los datos asociados permanezcan en la región geográfica designada (EE. UU. o UE). Para clientes del gobierno estadounidense, la infraestructura cumple los estándares FedRAMP Moderate. Un habilitador crítico para sectores regulados, administraciones públicas y empresas con requisitos de soberanía de datos.

→ Changelog: Copilot data residency in US + EU and FedRAMP compliance now available

2. Copilot SDK en vista previa pública

El SDK de Copilot expone el mismo motor de producción que impulsa el agente en la nube, disponible en cinco lenguajes (Node.js, Python, Go, .NET y Java). Las organizaciones pueden integrar capacidades de IA agéntica directamente en sus aplicaciones y servicios internos, sin necesidad de construir su propia capa de orquestación.

→ Changelog: Copilot SDK in public preview

3. Selección de modelo para los agentes Claude y Codex en github.com

Al iniciar una tarea con los agentes de codificación de terceros (Claude de Anthropic y Codex de OpenAI), los usuarios pueden seleccionar el modelo de IA más adecuado: Claude Opus 4.6, GPT-5.4 y otros modelos de última generación. Mayor control y acceso a las capacidades más avanzadas disponibles, incluidas en la suscripción existente de Copilot.

→ Changelog: Model selection for Claude and Codex agents on github.com

4. Las métricas de uso incluyen ahora los usuarios activos del agente en la nube

La API de métricas de uso de Copilot incorpora nuevos contadores de usuarios activos del agente (diarios, semanales y mensuales) a nivel de empresa y organización. Junto con los nuevos indicadores de ciclo de vida de pull requests, ofrece una visión completa del valor generado por Copilot en los flujos de desarrollo.

→ Changelog: Copilot usage metrics now aggregate Copilot cloud agent active user counts

5. Controles de runner para el agente en la nube a nivel de organización

Los administradores pueden establecer un runner predeterminado para todas las tareas del agente en la nube —incluyendo runners propios o de gran capacidad— y bloquear la configuración a nivel de repositorio. Garantiza la coherencia operativa y el cumplimiento de las políticas de infraestructura necesarios a escala empresarial.

→ Changelog: Organization runner controls for Copilot cloud agent

📌 También en esta edición

  • El agente en la nube de Copilot firma sus commits — Los commits aparecen como Verified en GitHub; el agente ahora funciona en repositorios con firma de commits obligatoria habilitada.
  • Instrucciones personalizadas de organización para Copilot disponibles con carácter general — Los administradores pueden definir instrucciones predeterminadas que guían el comportamiento de Copilot en todos los repositorios de la organización.
  • Métricas de fusión de PR revisados por Copilot en la API de métricas de uso — Dos nuevos indicadores para medir el impacto de las revisiones de código de Copilot en el ciclo de vida de los pull requests.
  • Resolución de conflictos de fusión en tres clics con el agente en la nube — Un botón «Fix with Copilot» en github.com delega la resolución de conflictos al agente, que verifica pruebas y hace el push de forma autónoma.
