Biweekly #2 · Del 23 de marzo al 6 de abril de 2026 · Decision-Makers Edition

🎯 El hilo de esta edición

Durante estas dos semanas, GitHub ha dado un paso decisivo para convertir Copilot en un agente de desarrollo autónomo a escala empresarial. El agente en la nube ahora investiga, planifica y escribe código; las organizaciones cuentan con controles centralizados de infraestructura y red; y las instrucciones personalizadas alcanzan disponibilidad general. La transición de asistente a agente autónomo cobra forma real.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Acción requerida — Estos tres modelos fueron retirados el 1 de abril. Actualiza tus integraciones para usar GPT-5.3-Codex. Los administradores deben habilitar el modelo alternativo en las políticas de modelos de Copilot.
  • Próxima retirada de Claude Sonnet 4 en GitHub Copilot — Fecha límite: 1 de mayo de 2026 — Migra a Claude Sonnet 4.6 antes de esa fecha. Los administradores de Copilot Enterprise deben verificar la disponibilidad del modelo alternativo en la configuración de políticas de modelos.

🚀 Pasa a la acción

  • Activa el agente en la nube de Copilot para tu organización desde la configuración de Copilot Business o Enterprise.
  • Define instrucciones personalizadas a nivel de organización para estandarizar el comportamiento de Copilot en todos los repositorios.
  • Configura los controles de runner y el firewall del agente de Copilot cloud agent para toda la organización.
  • Consulta las métricas de uso de Copilot para visualizar la adopción del agente y la CLI en tu organización.
  • ¿Tienes preguntas o necesitas ayuda? Contacta con tu equipo de GitHub, estamos aquí para apoyarte.

🏆 Top 5 de actualizaciones

1. El agente en la nube de Copilot ya investiga, planifica y escribe código

Copilot cloud agent ya no se limita a abrir pull requests. Ahora puede investigar tu codebase, generar un plan de implementación para revisión previa y trabajar en una rama sin crear PR hasta que estés listo. Un agente de desarrollo verdaderamente autónomo, bajo tu control.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. Controles de infraestructura y red para el agente en la nube a nivel de organización

Los administradores pueden ahora definir el runner por defecto del agente y bloquearlo para todos los repositorios, y gestionar la configuración del firewall de red de forma centralizada. Copilot cloud agent está listo para despliegues empresariales a gran escala con las garantías de seguridad necesarias.

Sources:
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

3. Instrucciones personalizadas de organización ya en disponibilidad general

Los administradores de Copilot Business y Copilot Enterprise pueden definir instrucciones predeterminadas que guían el comportamiento de Copilot en todos los repositorios: en Copilot Chat, en code review y en el agente en la nube. Coherencia operativa a escala, sin configuración repositorio por repositorio.

→ Changelog: Copilot organization custom instructions are generally available

4. Copilot SDK en vista previa pública: construye sobre el agente de Copilot

El SDK de Copilot expone el mismo runtime de agente que impulsa Copilot cloud agent y Copilot CLI, disponible en cinco lenguajes. Equipos de plataforma y partners pueden integrar las capacidades agénticas de Copilot directamente en sus aplicaciones y servicios. Una apuesta estratégica por el ecosistema.

→ Changelog: Copilot SDK in public preview

5. Métricas de uso de Copilot: visibilidad total sobre el agente y la CLI

Las métricas ahora identifican qué usuarios activan el agente en la nube y, en los informes de organización, el detalle de actividad por usuario en la CLI. Los responsables pueden ver la adopción completa de Copilot más allá del IDE para justificar la inversión y planificar la expansión del despliegue.

Sources:
→ Changelog: Copilot usage metrics now identify active Copilot coding agent users
→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

  • Crea issues desde Slack con Copilot — Describe trabajo en lenguaje natural en Slack y @GitHub crea issues estructurados directamente en tu repositorio, con soporte para sub-issues.
