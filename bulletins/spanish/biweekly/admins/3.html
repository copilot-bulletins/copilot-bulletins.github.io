Biweekly #3 · Del 1 al 15 de abril de 2026 · Admins Edition

🎯 El hilo de esta edición

Esta quincena refuerza el control administrativo de Copilot en tres frentes: residencia de datos y cumplimiento FedRAMP para requisitos regulatorios, controles centralizados del agente en la nube para una gobernanza coherente, y métricas de uso ampliadas que diferencian la adopción activa de la cobertura pasiva. Las organizaciones ya tienen las herramientas para desplegar Copilot con confianza a escala.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Acción requerida — Habilita GPT-5.3-Codex como alternativa en las políticas de modelos de Copilot de tu empresa u organización; los modelos sustituidos fueron retirados el 1 de abril de 2026.

🚀 Pasa a la acción

  • Activa la residencia de datos (US/EU) y las políticas de cumplimiento FedRAMP desde la configuración de datos de Copilot de tu empresa u organización.
  • Habilita GPT-5.3-Codex en tus políticas de modelos de Copilot para reemplazar los modelos GPT-5.1 Codex retirados.
  • Revisa la adopción de Copilot code review diferenciando usuarios activos y pasivos desde las métricas de uso de Copilot.
  • Configura los controles de runner y el firewall del agente en la nube a nivel de organización para garantizar una configuración coherente.
  • ¿Preguntas o necesitas ayuda? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 5 de actualizaciones

1. Residencia de datos en US + EU y cumplimiento FedRAMP ya disponibles

GitHub Copilot ahora asegura que todo el procesamiento de inferencia permanezca en la región designada (EE. UU. o UE), con opción de certificación FedRAMP Moderate para el sector público. Los administradores deben activar esta política explícitamente; implica un incremento del 10 % en el multiplicador del modelo.

→ Changelog: Copilot data residency in US + EU and FedRAMP compliance now available

2. Métricas de uso identifican usuarios activos y pasivos en revisión de código

Las métricas de uso de Copilot ahora diferencian entre usuarios que interactúan activamente con la revisión de código (asignando Copilot como revisor, aplicando sugerencias) y aquellos cuyas PRs son revisadas automáticamente sin intervención. Esto permite medir el nivel real de adopción, no solo la cobertura.

→ Changelog: Copilot usage metrics now identify active and passive Copilot code review users

3. Métricas de PRs revisadas por Copilot: tasa de fusión y tiempo de ciclo

La API de métricas de uso suma dos nuevos indicadores sobre Copilot code review: el total de PRs fusionadas con revisión de Copilot y el tiempo mediano hasta la fusión de esas PRs. Complementa las métricas existentes de PRs creadas por el agente para una visibilidad completa del ciclo de vida.

→ Changelog: Copilot-reviewed pull request merge metrics now in the usage metrics API

4. Controles de runner a nivel de organización para el agente en la nube

Los administradores de organización ahora pueden establecer un runner predeterminado para todas las ejecuciones del agente en la nube y bloquear la configuración a nivel de repositorio. Esto facilita desplegar el agente de forma coherente en toda la organización, ya sea con runners más potentes o auto-alojados.

→ Changelog: Organization runner controls for Copilot cloud agent

5. Configuración del firewall del agente en la nube a nivel de organización

Los administradores de organización ahora controlan el firewall del agente en la nube para todos los repositorios: activar o desactivar el firewall y la lista permitida recomendada, añadir entradas a una lista personalizada de toda la organización, y controlar si los repositorios pueden añadir sus propias reglas.

→ Changelog: Organization firewall settings for Copilot cloud agent

📌 También en esta edición

  • Actividad de Copilot CLI incluida en los totales de métricas de uso — Los totales de la API de métricas ahora combinan actividad de IDE y CLI; revisa tus dashboards si usabas estos campos como indicadores exclusivos de IDE.
  • El agente en la nube de Copilot firma sus commits — Los commits del agente ahora aparecen como "Verified", lo que permite usarlo en repositorios con la regla de commits firmados obligatoria activada.
  • Selección de modelo para los agentes Claude y Codex en github.com — Ahora es posible elegir el modelo concreto al utilizar los agentes de terceros (Claude, Codex) directamente en github.com.
