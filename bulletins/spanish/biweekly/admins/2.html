Biweekly #2 · Del 23 de marzo al 6 de abril de 2026 · Admins Edition

🎯 El hilo de esta edición

Las últimas dos semanas representan un avance clave en el control administrativo de Copilot. GitHub ha centralizado la configuración del Copilot cloud agent a nivel organización: runners, cortafuegos e instrucciones personalizadas son ahora gestionables de forma unificada en todos los repositorios, dando a los administradores las herramientas para desplegar el agente con las garantías de seguridad requeridas.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated — Acción requerida — Estos modelos se retiraron el 1 de abril de 2026. Los administradores deben habilitar GPT-5.3-Codex a través de las políticas de modelos de la organización para garantizar la continuidad del servicio.
  • Upcoming deprecation of Claude Sonnet 4 in GitHub Copilot — Fecha límite: 1 de mayo de 2026 — Habilitar Claude Sonnet 4.6 como alternativa desde la configuración de políticas de modelos antes de la fecha de retirada.

🚀 Pasa a la acción

  • Establece el runner predeterminado del cloud agent e impide anulaciones a nivel repositorio en la configuración de runners de tu organización.
  • Configura el cortafuegos del agente a nivel organización para proteger contra la exfiltración de datos en los ajustes del cortafuegos.
  • Define instrucciones personalizadas globales para alinear el comportamiento de Copilot con los estándares de tu organización en Ajustes > Copilot > Instrucciones personalizadas.
  • Consulta las métricas de uso del CLI de Copilot por usuario en los informes de métricas de uso para identificar patrones de adopción y planificar la expansión.
  • ¿Tienes preguntas o necesitas soporte? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 5 de actualizaciones

1. Control de runners a nivel organización para el cloud agent

Los administradores pueden definir el runner predeterminado para todas las sesiones del cloud agent en su organización y bloquear su modificación por los repositorios. Permite estandarizar el entorno de ejecución, usar runners grandes o self-hosted, y garantizar un comportamiento uniforme.

→ Changelog: Organization runner controls for Copilot cloud agent

2. Configuración del cortafuegos del cloud agent a nivel organización

Los administradores de organización pueden ahora gestionar el cortafuegos del cloud agent de forma centralizada: activar o desactivar el allowlist recomendado, añadir entradas personalizadas globales y controlar si los repositorios pueden añadir sus propias excepciones.

→ Changelog: Organization firewall settings for Copilot cloud agent

3. Instrucciones personalizadas de organización disponibles de forma general (GA)

Disponibles con carácter general para Copilot Business y Enterprise, las instrucciones de organización permiten establecer directrices globales que guían el comportamiento de Copilot en chat, revisiones de código y sesiones del cloud agent en toda la organización.

→ Changelog: Copilot organization custom instructions are generally available

4. Métricas de CLI de Copilot por usuario en informes de organización

Los administradores pueden ver qué usuarios utilizan activamente el CLI de Copilot, con detalles de sesiones, solicitudes, tokens consumidos y versión instalada, en informes de 1 y 28 días. Facilita la planificación de la adopción y la asignación de costes.

→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

5. El Copilot cloud agent ahora firma sus commits

Los commits realizados por el cloud agent aparecen como "Verified" en GitHub. Esto permite usarlo en repositorios con la regla "Require signed commits" activada, eliminando una restricción anterior para equipos con políticas estrictas de integridad del código.

→ Changelog: Copilot cloud agent signs its commits
