Biweekly #1 · Del 2 al 16 de marzo de 2026 · Admins Edition

🎯 El hilo de esta edición

Las últimas dos semanas refuerzan la gobernanza de los agentes de IA. GitHub ha lanzado herramientas para monitorizar sesiones agénticas con filtros avanzados, controlar la aprobación de workflows de Actions, aplicar políticas mediante hooks de agente y gestionar el acceso a nuevos modelos como GPT-5.4. El hilo común: más autonomía para los agentes, más control para el administrador.

🚀 Pasa a la acción

  • Habilita la política de GPT-5.4 en los ajustes de Copilot de tu empresa para que tus usuarios puedan acceder al nuevo modelo.
  • Revisa los nuevos filtros del panel de control de agentes para supervisar la actividad agéntica en tu empresa.
  • Configura la aprobación de workflows del coding agent según el nivel de confianza de cada repositorio.
  • Explora los hooks de agente para automatizar la aplicación de políticas durante las sesiones agénticas.
  • ¿Preguntas o necesitas ayuda? Contacta a tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 5 de actualizaciones

1. Nuevos filtros de sesión en el panel de control de agentes

El panel de AI Controls de la empresa incluye ahora filtros por estado, repositorio y usuario para sesiones agénticas. Esto permite supervisar la actividad de los agentes de forma granular y garantizar el cumplimiento de las políticas de la organización.

→ Changelog: Discover and manage agent activity with new session filters

2. Aprobación opcional de workflows de Actions para el coding agent

Los administradores de repositorio pueden ahora omitir la aprobación humana de workflows de GitHub Actions cuando el coding agent envía cambios. Ideal para acelerar la iteración en repositorios de confianza, manteniendo el requisito por defecto en entornos más sensibles.

→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

3. GPT-5.4 disponible en GitHub Copilot

El nuevo modelo de OpenAI ya está disponible en todos los clientes de Copilot. Los administradores de Copilot Business y Enterprise deben habilitar la política de GPT-5.4 en los ajustes para que sus usuarios puedan acceder al modelo.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. Mejoras agénticas en GitHub Copilot para JetBrains

Agentes personalizados, sub-agentes y agente de plan son ahora GA. Los hooks de agente (preview) permiten aplicar políticas en puntos clave de la sesión. Importante: los administradores de Business/Enterprise deben habilitar la política de "Editor preview features" para hooks.

→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs

5. VS Code v1.110 — Control y extensibilidad de agentes

Hooks de agente para aplicar políticas, auto-aprobación con sandboxing de terminal y compactación de contexto para sesiones largas. Los administradores ganan herramientas para definir límites claros en la ejecución autónoma de agentes sin frenar la productividad.

→ Changelog: GitHub Copilot in Visual Studio Code v1.110 - February release
