Biweekly #2 · Del 23 de marzo al 6 de abril de 2026 · Devs Edition

🎯 El hilo de esta edición

Las últimas dos semanas consolidan un patrón claro: GitHub Copilot evoluciona hacia un ecosistema agéntico más compositivo y extensible. Desde la ejecución paralela en la CLI hasta un SDK público para integrar las capacidades del agente en aplicaciones propias, el foco está en multiplicar la productividad del desarrollador mediante automatización coordinada.

⚠️ Atención

  • GPT-5.1 Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini deprecados — Deprecado: 1 de abril de 2026 — Actualiza tus flujos de trabajo e integraciones para utilizar GPT-5.3-Codex u otros modelos compatibles.

🚀 Pasa a la acción

  • Usa /fleet en Copilot CLI para distribuir tareas entre múltiples agentes en paralelo — aprende cómo en la guía de /fleet.
  • Explora el SDK de Copilot en preview pública para integrar capacidades agénticas en tus propias herramientas (Node.js, Python, Go, .NET y Java).
  • Migra a GPT-5.3-Codex si aún usas los modelos GPT-5.1-Codex (retirados el 1 de abril de 2026).
  • ¿Tienes preguntas o necesitas ayuda? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 5 de actualizaciones

1. /fleet en Copilot CLI: múltiples agentes en paralelo

/fleet es un nuevo comando de Copilot CLI que orquesta varios subagentes de forma simultánea. El orquestador descompone tu objetivo en tareas independientes y las ejecuta en paralelo en diferentes partes del código, reduciendo el tiempo en refactorizaciones y features multi-archivo.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

2. SDK de Copilot en preview pública

El SDK de Copilot ya está disponible en preview pública para cinco lenguajes: Node.js, Python, Go, .NET y Java. Expone el mismo runtime de agente que impulsa Copilot cloud agent y Copilot CLI, con invocación de herramientas, streaming, operaciones de archivos y sesiones multi-turno listas para usar.

→ Changelog: Copilot SDK in public preview

3. GitHub Copilot en Visual Studio — actualización de marzo

La actualización de marzo trae agentes personalizados (.agent.md), skills reutilizables y la herramienta find_symbol para navegación semántica en modo agente. También incluye profiling con Copilot desde Test Explorer y sugerencias inteligentes en Watch windows durante la depuración.

→ Changelog: GitHub Copilot in Visual Studio — March update

4. Copilot cloud agent firma sus commits

Copilot cloud agent ahora firma todos sus commits, que aparecen como "Verified" en GitHub. Esto significa que el agente ya funciona en repositorios con la regla de rama "Require signed commits" activada, eliminando uno de los principales bloqueos para su adopción.

→ Changelog: Copilot cloud agent signs its commits

5. Controles de runner para Copilot cloud agent a nivel de organización

Los administradores pueden ahora definir el runner predeterminado para Copilot cloud agent en toda la organización —incluidos runners de mayor capacidad o auto-hospedados— y bloquear la configuración para uniformar el entorno de ejecución del agente.

→ Changelog: Organization runner controls for Copilot cloud agent
