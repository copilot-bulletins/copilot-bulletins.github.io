Biweekly #3 · Del 1 al 15 de abril de 2026 · Devs Edition

🎯 El hilo de esta edición

Las últimas dos semanas consolidan un patrón claro: Copilot se expande hacia donde trabajan los desarrolladores. VS Code recibe modos autónomos de próxima generación, la CLI se puede controlar desde cualquier dispositivo, y los agentes cloud ya resuelven conflictos de fusión directamente en los pull requests. El ecosistema se abre además con BYOK y un SDK público.

⚠️ Atención

  • GPT-5.1-Codex, GPT-5.1-Codex-Max y GPT-5.1-Codex-Mini retirados — Fecha de baja: 1 de abril de 2026 — Actualiza tus workflows e integraciones para usar GPT-5.3-Codex como alternativa; consulta los modelos disponibles.
  • Opus 4.6 Fast retirado de Copilot Pro+ — Acción requerida — Cambia a Opus 4.6 como alternativa de capacidades similares; si alcanzas restricciones de tasa, consulta la documentación de rate limiting.

🚀 Pasa a la acción

  • Activa las sesiones remotas con copilot --remote para controlar tu agente desde cualquier dispositivo: consulta la documentación de acceso remoto.
  • Conecta tu propio proveedor de LLM o modelos locales con la guía de configuración BYOK.
  • Explora el Copilot SDK (public preview) para integrar las capacidades de agente en tus propias aplicaciones.
  • Actualiza tus integraciones que usen GPT-5.1-Codex a GPT-5.3-Codex.
  • ¿Preguntas o necesitas ayuda? Contacta con tu equipo de GitHub, estamos aquí para ayudarte.

🏆 Top 5 de actualizaciones

1. VS Code Copilot — versiones de marzo: modo Autopilot y más

VS Code Copilot estrena el modo Autopilot (preview): aprueba sus propias acciones, reintenta errores y trabaja de forma autónoma hasta completar la tarea. También llegan subagentes anidados, depuración en el navegador integrado, soporte de imágenes y vídeo en el chat, y personalizaciones por monorepositorios.

→ Changelog: GitHub Copilot in Visual Studio Code, March Releases

2. Sesiones de la CLI en remoto desde web y móvil

copilot --remote hace streaming de tu sesión de CLI en tiempo real hacia GitHub.com o la app móvil. Desde otro dispositivo puedes enviar mensajes, aprobar permisos, revisar planes y mantener el agente activo sin estar frente a tu máquina.

→ Changelog: Remote control CLI sessions on web and mobile in public preview

3. Resuelve conflictos de fusión en tres clics con el agente cloud

El nuevo botón "Fix with Copilot" aparece en el cuadro de fusión de cualquier pull request con conflictos. El agente cloud analiza los cambios, resuelve los conflictos y verifica que la compilación y los tests siguen pasando, todo desde su propio entorno cloud.

→ Changelog: Fix merge conflicts in three clicks with Copilot cloud agent

4. Selección de modelo para los agentes Claude y Codex en github.com

Claude y Codex ahora permiten elegir el modelo al lanzar una tarea en github.com, igual que el agente cloud de Copilot. Para Claude: Sonnet 4.6, Opus 4.6, Sonnet 4.5 y Opus 4.5. Para Codex: GPT-5.2-Codex, GPT-5.3-Codex y GPT-5.4.

→ Changelog: Model selection for Claude and Codex agents on github.com

5. Copilot CLI con soporte BYOK y modelos locales

La CLI ahora se conecta a Azure OpenAI, Anthropic o cualquier endpoint compatible con OpenAI —incluyendo Ollama y vLLM— como proveedor de modelos. Con COPILOT_OFFLINE=true se habilita un modo totalmente air-gapped, y la autenticación de GitHub es opcional cuando aportas tus propias credenciales.

→ Changelog: Copilot CLI now supports BYOK and local models
