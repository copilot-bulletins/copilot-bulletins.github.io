Monthly #1 · March 2026 · Admins Edition

🎯 This edition's thread

This month, GitHub Copilot's enterprise governance story reached a milestone. AI Controls and the agent control plane are now generally available alongside fully GA usage metrics with expanded telemetry. Combined with new multi-agent support and agentic code review, admins have a comprehensive toolkit to govern, monitor, and scale AI-assisted development across their organizations.

⚠️ Heads up

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline: March 26 (Gemini 3 Pro) / April 1 (GPT-5.1 family) — Ensure alternative models are enabled in your Copilot model policies before these dates.

🚀 Take action

  • Enable GPT-5.4 and review model policies in your Copilot settings ahead of upcoming deprecations.
  • Enable Claude and Codex agents for your organization under Enterprise AI Controls.
  • If using self-hosted runners, complete one-time setup for agentic code review.
  • Explore the new Copilot usage metrics dashboards to track adoption and engagement across your enterprise.
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 5 updates of the month

1. Enterprise AI Controls and agent control plane reach GA

Enterprise admins can now centrally govern all AI agents from a single AI Controls view. Monitor sessions with filters by status, repository, and user, enforce custom agent policies via push rules, and access detailed audit logs for all agentic activity.

→ Changelog: Enterprise AI Controls & agent control plane now generally available

2. Copilot usage metrics goes GA with expanded telemetry

Copilot usage metrics is now generally available with dashboards and APIs at enterprise, organization, and user levels. This month also added Copilot CLI telemetry, plan mode tracking, and consistent EMU usernames — giving admins a complete picture of Copilot adoption.

Sources:
→ Changelog: Copilot metrics is now generally available
→ Changelog: Copilot usage metrics now includes enterprise-level GitHub Copilot CLI activity
→ Changelog: Copilot usage metrics now includes user-level GitHub Copilot CLI activity
→ Changelog: Copilot metrics now includes plan mode

3. Claude and Codex agents available for Business and Pro plans

Claude by Anthropic and OpenAI Codex are now included with Copilot Business and Pro at no extra cost. Admins enable them at enterprise and organization levels through AI Controls, with unified governance, shared memory, and centralized audit logging across all agents.

→ Changelog: Claude and Codex now available for Copilot Business & Pro users

4. Copilot code review upgrades to agentic architecture

Copilot code review now uses agentic tool calling to gather broader repository context, delivering higher-quality findings with less noise. The feature is GA across all paid plans. Organizations using self-hosted runners need to complete a one-time setup to receive agentic reviews.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Blog: 60 million Copilot code reviews and counting

5. New admin controls for Copilot governance at scale

Admins gain finer control over Copilot: a new repository setting lets you skip workflow approval for the coding agent, the Content Exclusion REST API enables programmatic governance at scale, and enhanced session filters improve agent activity monitoring.

Sources:
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
→ Changelog: Copilot Content Exclusion REST API in public preview
→ Changelog: Discover and manage agent activity with new session filters
