Monthly #2 · April 2026 · Admins Edition

🎯 This edition's thread

March and April 2026 saw a significant expansion of organization-level governance controls for Copilot cloud agent — covering runner configuration, firewall policies, commit signing, and usage analytics. Combined with custom instructions reaching GA, this month marks a clear maturation point for enterprise Copilot rollouts, giving admins the tools to standardize, secure, and measure deployments at scale.

⚠️ Heads up

  • GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated — Deadline: April 1, 2026 (already retired) — Review your model policies and enable GPT-5.3-Codex as the suggested replacement; no action is needed to remove the deprecated models.

🚀 Take action

  • Set your org-level runner defaults and lock configurations for Copilot cloud agent in your organization Copilot cloud agent settings
  • Configure org-wide firewall policies and custom allowlists for cloud agent via Customizing the agent firewall
  • Define organization custom instructions to standardize Copilot behavior across all repos in your Copilot custom instructions settings
  • Verify your model policies include GPT-5.3-Codex to replace the now-deprecated GPT-5.1-Codex variants
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 5 updates of the month

1. Organization-wide runner and firewall controls for Copilot cloud agent

Org admins can now set default GitHub Actions runners and lock them org-wide, while also managing the agent firewall across all repositories. This enables consistent, scalable cloud agent deployments — including self-hosted runners and org-level allowlists to guard against data exfiltration.

Sources:
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

2. Copilot organization custom instructions are generally available

Org admins on Business and Enterprise plans can now set default natural language instructions that guide Copilot's behavior across all repositories. Applied to Chat on GitHub.com, code review, and cloud agent, this ensures consistent standards org-wide without per-repository setup.

→ Changelog: Copilot organization custom instructions are generally available

3. Copilot cloud agent signs its commits

Copilot cloud agent now signs every commit it makes, showing as Verified on GitHub. This removes a key blocker for security-conscious organizations with the "Require signed commits" branch protection rule or ruleset enabled — the agent can now operate in those repositories.

→ Changelog: Copilot cloud agent signs its commits

4. Full per-user CLI usage visibility in organization reports

Organization admins now have per-user Copilot CLI breakdowns in org reports, completing coverage across enterprise, user, and org levels. Track active CLI users, session and token counts, and CLI version distribution — essential data for rollout planning, cost allocation, and upgrade governance.

→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

5. Research, plan, and code with Copilot cloud agent

Copilot cloud agent is no longer limited to pull-request workflows. Teams can now start deep research sessions, generate implementation plans for review before any code is written, and work on branches without auto-creating a PR — giving admins a more flexible feature to roll out.

→ Changelog: Research, plan, and code with Copilot cloud agent
