🏆 Top 5 updates of the month

1. Copilot Cloud Agent: Research, Plan, and Code Autonomously

Copilot cloud agent can now research codebases, generate implementation plans for review before any code is written, and work independently on a branch without opening a pull request. Organizations can delegate entire development workstreams—from discovery to implementation—directly to Copilot, reviewing the diff and iterating before a PR is created.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. Enterprise-Wide Governance Controls for Copilot Cloud Agent

Three new organization-level settings give admins centralized control: custom instructions are now GA (guiding Copilot behavior across all repos), plus new runner controls and firewall policies. Teams can enforce consistent AI behavior, compute standards, and network guardrails across the entire organization without per-repository configuration.

Sources:
→ Changelog: Copilot organization custom instructions are generally available
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Organization firewall settings for Copilot cloud agent

3. State-of-the-Art Models with an Enterprise LTS Commitment

GPT-5.4, OpenAI's latest agentic model, is now GA on all paid Copilot plans with enhanced reasoning and multi-step task execution. GitHub also launched an enterprise Long-Term Support program: GPT-5.3-Codex is the first LTS model, guaranteed available for 12 months, giving security and compliance teams the stability they need for internal review processes.

Sources:
→ Changelog: GPT-5.4 is generally available in GitHub Copilot
→ Changelog: GPT-5.3-Codex long-term support in GitHub Copilot

4. Copilot SDK: Embed Agentic AI into Your Enterprise Workflows

The Copilot SDK is now in public preview, exposing the agent runtime that powers Copilot cloud agent. Available in 5 languages (Node.js, Python, Go, .NET, Java), it lets engineering teams embed Copilot-grade AI directly into internal tools and platforms, with custom tool definitions, Bring Your Own Key support, and fine-grained permission controls.

→ Changelog: Copilot SDK in public preview

5. Trust and Visibility: Signed Commits and Expanded Usage Metrics

Copilot cloud agent now signs every commit it makes—displayed as "Verified" on GitHub—enabling its use in repositories with branch protection rules requiring signed commits. Alongside this, usage metrics now track per-user coding agent adoption, resolve auto model selection to actual models used, and include Copilot CLI activity per user.
