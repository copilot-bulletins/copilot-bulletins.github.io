🏆 Top 5 updates of the month

1. Copilot SDK now in public preview

The GitHub Copilot SDK exposes the same production-tested agent runtime that powers Copilot cloud agent and Copilot CLI. Available in Node.js/TS, Python, Go, .NET, and Java, it lets you embed agentic AI—with custom tools, streaming, and BYOK—directly into your own applications.

→ Changelog: Copilot SDK in public preview

2. /fleet: Run multiple agents in parallel in Copilot CLI

The new /fleet slash command dispatches multiple sub-agents simultaneously across your codebase. An orchestrator breaks your objective into independent work items, runs them in parallel, and synthesizes results—cutting time on large refactors, cross-file features, and documentation updates.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

3. Visual Studio March update: Custom agents, skills, and profiling

The Visual Studio 2026 March update adds custom .agent.md agents with full workspace and MCP access, reusable agent skills, and a find_symbol tool for language-aware code navigation. New profiling features let Copilot analyze test performance and suggest targeted optimizations inline while debugging.

→ Changelog: GitHub Copilot in Visual Studio — March update

4. Copilot cloud agent now signs its commits

Copilot cloud agent signs every commit it makes, displaying as Verified on GitHub. This removes a key compatibility barrier: the agent now works in repositories with the "Require signed commits" branch protection rule or ruleset enabled, where it was previously blocked entirely.

→ Changelog: Copilot cloud agent signs its commits

5. Org-level runner and firewall controls for Copilot cloud agent

Organization admins can set a default runner for all repos—and optionally lock it—enabling consistent use of larger or self-hosted runners at scale. The agent firewall is also now configurable at the org level, with controls to toggle access and add custom allowlist entries.
