Biweekly #3 · March 10 to March 24, 2026 · Admins Edition

🎯 This edition's thread

This fortnight, governance and auditability of AI-assisted development took center stage. GitHub introduced a long-term support (LTS) model commitment, granular admin control over coding agent validation tooling, and improved traceability of agent-authored commits — all signaling a maturing enterprise control plane around Copilot. Admins now have clearer levers to configure, audit, and comply as AI adoption scales.

⚠️ Heads up

  • GPT-5.3-Codex long-term support in GitHub Copilot — Deadline: May 17, 2026 — GPT-5.3-Codex will automatically become the default base model for all Copilot Business and Enterprise organizations. Review your model management policies now and contact your account team if your organization needs more time before the transition.

🚀 Take action

  • Configure which security tools (CodeQL, secret scanning, Copilot code review) the coding agent runs by visiting Copilot → Coding agent in your repository settings
  • Enable Gemini 3.1 Pro and GPT-5.4 mini in your organization's Copilot model policies to give users access to the latest models
  • Enable Dependabot malware alerting under Settings → Code security → Dependabot to detect malicious npm packages before they affect your organization's repositories
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 5 updates

1. Long-Term Support for Copilot Enterprise Models

GitHub has introduced LTS models for Copilot Business and Copilot Enterprise, starting with GPT-5.3-Codex. Designated LTS models remain available for a full 12 months, giving organizations the stability needed to complete internal security and safety reviews without forced model migrations.

→ Changelog: GPT-5.3-Codex long-term support in GitHub Copilot

2. Clearer Model-Level Usage Metrics for Compliance

Copilot usage metrics now resolve the generic "Auto" label to the actual AI model used — across the REST API and dashboard. Admins get a complete, accurate breakdown of model usage at the enterprise, org, and user levels, directly supporting compliance and audit reporting workflows.

→ Changelog: Copilot usage metrics now resolve auto model selection to actual models

3. Admin Control Over Coding Agent Validation Tools

Repository admins can now configure which security and quality tools the Copilot coding agent runs — including CodeQL, secret scanning, and Copilot code review — from the Copilot settings in repository configuration. Useful when certain checks create unacceptable latency for specific projects.

→ Changelog: Configure Copilot coding agent's validation tools

4. Secret Scanning in AI Coding Agents via GitHub MCP Server

The GitHub MCP Server can now run secret scanning on code changes before commit, in any MCP-compatible IDE or AI coding agent. In public preview for repositories with GitHub Secret Protection enabled, this extends shift-left security to AI-assisted development workflows.

→ Changelog: Secret scanning in AI coding agents via the GitHub MCP Server

5. Audit Trail: Trace Agent Commits Back to Session Logs

Every commit from Copilot coding agent now includes an Agent-Logs-Url trailer in the commit message, providing a permanent link to the full agent session logs. This makes it straightforward to understand what Copilot did and why — directly from code review or during a post-incident audit.

→ Changelog: Trace any Copilot coding agent commit to its session logs
