🏆 Top 5 updates

1. Regional Data Residency and FedRAMP Compliance for GitHub Copilot

GitHub Copilot now keeps all inference and associated data within your designated US or EU region, with FedRAMP Moderate compliance for US government customers. Enterprise and org admins can opt in from Copilot settings; all GA features are supported. Note a 10% model-multiplier surcharge applies for data-resident and FedRAMP-certified requests.

→ Changelog: Copilot data residency in US + EU and FedRAMP compliance now available

2. Organization-Wide Firewall Controls for Copilot Cloud Agent

Organization admins can now centrally manage the Copilot cloud agent's network firewall across all repositories—toggling it on or off, managing the recommended allowlist, and adding org-level custom allowlist entries. Admins can also restrict whether repo admins are allowed to extend the allowlist, replacing fragmented per-repository configuration with a single governance layer.

→ Changelog: Organization firewall settings for Copilot cloud agent

3. Centralized Runner Controls for Copilot Cloud Agent

Organization admins can now set a default GitHub Actions runner type for Copilot cloud agent across all repositories and optionally lock the setting to prevent overrides. This enables consistent guardrails—for example, ensuring agent sessions always run on self-hosted runners with access to internal resources.

→ Changelog: Organization runner controls for Copilot cloud agent

4. Active vs. Passive Copilot Code Review Engagement in Usage Metrics

Usage metrics now distinguish whether a user actively engaged with Copilot code review (assigned reviews, applied suggestions) or passively had reviews run via an automated policy. Admins can now separate coverage from genuine adoption—moving from "how many users are covered" to "how many actually engage."

→ Changelog: Copilot usage metrics now identify active and passive Copilot code review users

5. Measure Copilot Code Review Impact via the Usage Metrics API

Two new fields in the usage metrics API track Copilot code review outcomes: total merged pull requests that received a Copilot review, and their median time-to-merge. Combined with existing coding-agent PR metrics, admins now have end-to-end visibility into how Copilot contributes across the full pull request lifecycle.

→ Changelog: Copilot-reviewed pull request merge metrics now in the usage metrics API
