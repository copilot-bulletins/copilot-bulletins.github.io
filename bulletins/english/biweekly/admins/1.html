Biweekly #1 · March 2 to March 16, 2026 · Admins Edition

🎯 This edition's thread

The past two weeks focused on giving administrators stronger control over the expanding agentic ecosystem. From enterprise-wide session monitoring with new filters to repository-level workflow approval settings and agent hooks for policy enforcement, GitHub is building the governance layer admins need to manage AI agents at scale while balancing security with developer velocity.

🚀 Take action

  • Enable GPT-5.4 for your organization in your Copilot policies.
  • Review your coding agent workflow approval settings to decide if auto-run makes sense for specific repositories.
  • Monitor agent activity across your enterprise using the new session filters in AI Controls.
  • Explore agent hooks to enforce coding standards and policies during agent sessions.
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 5 updates

1. Enterprise agent control plane gets granular session filters

The AI Controls view now lets enterprise owners filter agent sessions by status, repository, and user — on top of existing agent and organization filters. This makes it easier to audit agent activity, spot issues, and ensure compliance across your enterprise.

→ Changelog: Discover and manage agent activity with new session filters

2. Skip workflow approval for Copilot coding agent PRs

A new repository setting lets admins disable the manual "Approve and run workflows" step for Copilot coding agent pull requests. This speeds up iteration for trusted repositories but requires careful evaluation — workflows may access secrets and repository permissions.

→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

3. GPT-5.4 rolls out across GitHub Copilot

OpenAI's latest agentic coding model is now available in all Copilot clients. Copilot Business and Enterprise administrators must enable the GPT-5.4 policy in Copilot settings before their teams can use it. The model delivers improved reasoning for complex, multi-step tasks.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. Auto model selection GA in JetBrains — respects admin policies

Copilot auto model selection is now generally available in JetBrains IDEs. It dynamically routes to models like GPT-5.4, Sonnet 4.6, and Haiku 4.5 based on availability — while honoring all administrator model settings. Paid subscribers get a 10% discount on premium request multipliers when using auto.

→ Changelog: Copilot auto model selection is generally available in JetBrains IDEs

5. Agent hooks and policy controls land in JetBrains

Custom agents, sub-agents, and plan agent are now GA in JetBrains. Agent hooks (public preview) let teams run custom commands at key agent lifecycle events to enforce policies and automate workflows. Copilot Business and Enterprise admins must enable the "Editor preview features" policy for hooks.

→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs
