Admins now have greater clarity into actual model usage across their organizations, a permanent audit trail for AI-generated code, and new controls over coding agent security validation. The throughline: enterprise-grade oversight is expanding in step with broader AI adoption.

🏆 Top 5 updates

1. First LTS Model and a New Base Model on the Horizon

GitHub introduced a long-term support (LTS) program for Copilot models, with GPT-5.3-Codex as the first designated LTS model — guaranteed available for 12 months from its February 5, 2026 launch. GPT-5.3-Codex also becomes the new base model for all Copilot Business and Enterprise organizations by May 17, 2026.

→ Changelog: GPT-5.3-Codex long-term support in GitHub Copilot

2. Usage Metrics Now Show Real Models Behind Auto Mode

When auto model selection is active, usage data previously showed a generic "Auto" label. Now it resolves to the actual model name — across enterprise, org, and user reports. Admins get complete, accurate visibility into which models their teams are using for compliance and audit workflows.

→ Changelog: Copilot usage metrics now resolve auto model selection to actual models

3. CLI Activity Now Fully Visible in Org Usage Metrics

Organization-level CLI telemetry is now available, completing coverage alongside existing enterprise and user-level reports. Org admins can track daily active CLI users, session and request counts, and token usage — giving a complete picture of how teams consume Copilot from the command line.

→ Changelog: Copilot usage metrics now includes organization-level GitHub Copilot CLI activity

4. Repo Admins Now Control Coding Agent Security Validation

Repository admins can now configure which built-in validation tools run when Copilot coding agent writes code — CodeQL, secret scanning, the GitHub Advisory Database, and Copilot code review. All free, no GHAS license required, and enabled by default. Disable specific checks from repository Copilot settings when needed.

→ Changelog: Configure Copilot coding agent's validation tools

5. Agent Commits Now Link Directly to Session Logs

Every commit from Copilot coding agent now includes an Agent-Logs-Url trailer in the commit message, creating a permanent link from the commit back to the full session logs. Admins and reviewers can trace exactly what the agent did, when, and why — directly from code review or a future audit.

→ Changelog: Trace any Copilot coding agent commit to its session logs
