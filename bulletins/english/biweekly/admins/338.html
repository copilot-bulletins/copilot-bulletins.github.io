🏆 Top 5 updates

1. New session filters for enterprise agent monitoring

Enterprise admins can now filter agentic sessions by status, repository, and user — in addition to the existing agent and organization filters. This gives you granular visibility into how agents are being used across your enterprise, making it easier to audit activity and enforce compliance.

→ Changelog: Discover and manage agent activity with new session filters

2. GPT-5.4 is generally available — admins must enable it

OpenAI's latest agentic coding model is now available across all Copilot plans and clients. Copilot Business and Enterprise administrators must explicitly enable the GPT-5.4 policy in Copilot settings before their users can access it.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

3. Skip workflow approval for Copilot coding agent

A new repository setting lets admins optionally allow GitHub Actions workflows to run automatically when Copilot coding agent pushes changes — without waiting for human approval. By default, approval is still required, keeping security controls in place.

→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

4. Copilot code review reaches 60 million reviews

Copilot code review usage has grown 10X since launch, now accounting for one in five code reviews on GitHub. More than 12,000 organizations run it automatically on every pull request, using its agentic architecture to surface high-signal feedback.

→ Blog: 60 million Copilot code reviews and counting

5. Security architecture of Agentic Workflows deep-dive

GitHub published a detailed breakdown of how Agentic Workflows enforce security: defense in depth with isolated containers, zero-secret agents, staged and vetted writes via safe outputs, and comprehensive logging at every trust boundary. Essential reading for evaluating agentic AI governance.

→ Blog: Under the hood: Security architecture of GitHub Agentic Workflows
