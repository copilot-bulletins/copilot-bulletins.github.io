2. Copilot coding agent starts work 50% faster

Copilot coding agent has been optimized to start work 50% faster. Whether you're assigning an issue, prompting via the Agents tab, or mentioning @copilot in a PR comment, the wait before Copilot picks up the task is significantly shorter — tighter iteration loops, faster PR creation.

→ Changelog: Copilot coding agent now starts work 50% faster

3. Gemini 3.1 Pro reaches JetBrains, Xcode, and Eclipse

Gemini 3.1 Pro is now in public preview across all major Copilot-supported IDEs — including JetBrains, Xcode, and Eclipse — via the chat model picker in agent, ask, and edit modes. Business and Enterprise admins must enable the Gemini 3.1 Pro policy in Copilot settings to give their team access.

→ Changelog: Gemini 3.1 Pro is now available in JetBrains IDEs, Xcode, and Eclipse

4. Coding agent navigates your codebase with semantic search

Copilot coding agent now uses semantic code search to find relevant code by meaning — not just exact text matches. This helps it navigate unfamiliar codebases faster when it doesn't know precise names or patterns, reducing task completion time by ~2% with no configuration needed.

→ Changelog: Copilot coding agent works faster with semantic code search

5. Trace every agent commit back to its session logs

Every commit from Copilot coding agent now includes an Agent-Logs-Url trailer linking directly to the session logs. During code review or post-merge auditing, click the link to understand exactly what Copilot did and why — a permanent, navigable audit trail from commit to context.

→ Changelog: Trace any Copilot coding agent commit to its session logs
