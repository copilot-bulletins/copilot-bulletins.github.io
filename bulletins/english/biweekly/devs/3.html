Biweekly #3 · April 1 to April 15, 2026 · Devs Edition

🎯 This edition's thread

The past two weeks reveal a clear pattern in Copilot's evolution: greater developer autonomy and mobility. VS Code gained a fully autonomous Autopilot mode, the CLI shed its dependency on GitHub-hosted models with BYOK support, and agent sessions can now be monitored and steered remotely from any device. Copilot is becoming less tied to a single desk or tool.

⚠️ Heads up

  • GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated — Deprecated: April 1, 2026 — Update all workflows and integrations to use GPT-5.3-Codex; see the supported models documentation.

🏆 Top 5 updates

1. VS Code's March releases: Autopilot, browser debugging, and more

VS Code's March releases introduce Autopilot—a preview mode where agents approve their own actions and run to completion without manual intervention. Also new: browser debugging from the editor, image and video in chat, nested subagents, MCP servers shared with Copilot CLI sessions, and a unified chat customizations editor.

→ Changelog: GitHub Copilot in Visual Studio Code, March Releases

2. Copilot CLI now supports BYOK and local models

Copilot CLI now lets you bring your own model provider—Azure OpenAI, Anthropic, Ollama, or any OpenAI-compatible endpoint—instead of relying on GitHub-hosted routing. Set COPILOT_OFFLINE=true for fully air-gapped workflows. GitHub authentication is optional when using your own keys.

→ Changelog: Copilot CLI now supports BYOK and local models

3. Remote control CLI sessions from the web or mobile

Copilot CLI now streams your session to GitHub in real time. Start with copilot --remote, get a link and QR code, and monitor or steer your task from any browser or the GitHub Mobile app—without staying at the machine running the session.

→ Changelog: Remote control CLI sessions on web and mobile in public preview

4. Fix merge conflicts in three clicks with Copilot cloud agent

A new "Fix with Copilot" button now appears in the merge box on github.com when a PR has conflicts. Click it, submit the pre-populated comment, and Copilot resolves the conflicts, verifies the build passes, and pushes—all from its cloud-based development environment.

→ Changelog: Fix merge conflicts in three clicks with Copilot cloud agent

5. Pick your model for Claude and Codex coding agents

Model selection is now available for Claude and Codex third-party coding agents on github.com. Choose from Anthropic Sonnet and Opus variants (4.5 and 4.6) for Claude, or GPT-5.2-Codex, GPT-5.3-Codex, and GPT-5.4 for Codex—same model-picker experience as with Copilot cloud agent.

→ Changelog: Model selection for Claude and Codex agents on github.com
