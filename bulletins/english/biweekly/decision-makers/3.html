🏆 Top 5 updates

1. Copilot now meets enterprise data sovereignty and FedRAMP requirements

GitHub Copilot now keeps all inference and associated data within your designated US or EU region, meeting sovereignty requirements. US federal customers benefit from FedRAMP Moderate authorization across all Copilot features—agent mode, code review, cloud agent, and CLI—making enterprise-grade AI accessible to regulated industries.

→ Changelog: Copilot data residency in US + EU and FedRAMP compliance now available

2. Cross-model "Rubber Duck" closes 74.7% of the AI quality gap in Copilot CLI

Copilot CLI's Rubber Duck feature pairs a primary AI model with an independent reviewer from a different model family. Evaluations on SWE-Bench Pro show this approach closes 74.7% of the performance gap between Claude Sonnet and Opus, catching architectural errors and cross-file conflicts before they compound.

→ Blog: GitHub Copilot CLI combines model families for a second opinion

3. Claude and Codex agents on GitHub now offer full model selection

Teams using Claude or Codex coding agents on GitHub can now select specific model versions per task—from Claude Sonnet 4.5 to Opus 4.6, or GPT-5.2-Codex to GPT-5.4. This flexibility is included in existing Copilot subscriptions and requires only an admin policy change to enable.

→ Changelog: Model selection for Claude and Codex agents on github.com

4. /fleet dispatches multiple AI agents in parallel for large-scale tasks

The /fleet command in Copilot CLI decomposes large objectives—cross-module refactors, multi-file feature implementations, parallel documentation—into simultaneous workstreams executed by multiple subagents. Organizations can now compress hours of sequential AI work into parallel workflows, unlocking higher throughput on complex initiatives.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

5. Copilot CLI sessions can now be monitored and steered from web or mobile

Developers running long Copilot CLI sessions can now monitor and steer them from any browser or GitHub Mobile—reviewing plans, approving actions, and sending follow-up instructions without being at their terminal. This enables organizations to delegate complex background tasks that run unattended.

→ Changelog: Remote control CLI sessions on web and mobile in public preview
