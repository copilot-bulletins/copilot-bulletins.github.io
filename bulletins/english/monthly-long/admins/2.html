Monthly (Long) #2 · April 2026 · Admins Edition

🎯 This edition's thread

March brought significant maturation to Copilot's governance and observability story. Usage metrics now cover every Copilot surface — IDE, CLI, and cloud agent — giving administrators a complete, actionable picture of adoption across the organization. Organization-level controls were simultaneously reinforced: custom instructions reached general availability, and coding agent gained API-based repository scoping, audit-trail commits, org-wide runner defaults, and configurable validation tools.

Model policy also demands attention this month: Claude Sonnet 4 is scheduled for deprecation on May 1. The broader signal is that GitHub is systematically building enterprise-grade controls into every layer of the Copilot platform.

⚠️ Heads up

  • Upcoming deprecation of Claude Sonnet 4 in GitHub Copilot — Deadline: May 1, 2026 — Update workflows and integrations to use Claude Sonnet 4.6; Enterprise admins should verify the alternative model is enabled in their Copilot model policies.

🚀 Take action

  • Set organization-level custom instructions to standardize Copilot's behavior across all repositories via your organization Copilot custom instructions settings.
  • Review model policies and enable Claude Sonnet 4.6 before May 1 using the supported models documentation.
  • Enable Copilot cloud agent for your organization and set org-wide runner defaults — with optional lock — via the organization Copilot cloud agent runner controls.
  • Monitor Copilot adoption across IDE, CLI, and cloud agent surfaces using the Copilot usage metrics API.
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 8 updates of the month

1. Copilot usage metrics: Complete visibility across every surface

Usage metrics now cover CLI activity at org and per-user levels, identify active coding agent users, and resolve auto model selection to actual model names. Administrators gain a complete, accurate picture of Copilot adoption across IDE, CLI, and cloud agent — critical for adoption tracking and compliance reporting.

Sources:
→ Changelog: Copilot usage metrics now includes organization-level GitHub Copilot CLI activity
→ Changelog: Copilot usage metrics now resolve auto model selection to actual models
→ Changelog: Copilot usage metrics now identify active Copilot coding agent users
→ Changelog: Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

2. Organization custom instructions now generally available

Organization owners can set default instructions guiding Copilot's behavior across Copilot Chat, code review, and cloud agent sessions — for all repositories in the organization. Moving from preview to GA makes this a production-ready governance control for Copilot Business and Enterprise customers.

→ Changelog: Copilot organization custom instructions are generally available

3. Coding agent governance: Audit trail, runner controls, and API management

Every Copilot cloud agent commit now links to its full session log via an Agent-Logs-Url trailer — a permanent audit trail for code review and compliance. Admins can also set and lock org-wide runner defaults, manage repository access via the REST API, and configure validation tools and Actions workflow approvals.

Sources:
→ Changelog: Trace any Copilot coding agent commit to its session logs
→ Changelog: Organization runner controls for Copilot cloud agent
→ Changelog: Manage Copilot coding agent repository access via the API
→ Changelog: Configure Copilot coding agent's validation tools
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

4. Privacy Statement and Terms of Service updated

Starting April 24, GitHub will use interaction data from Copilot Free, Pro, and Pro+ users to improve AI models unless users opt out. Copilot Business and Enterprise customers are not affected. Admins should be aware of this change and communicate it to any Free or Pro users across their organization.

→ Changelog: Updates to our Privacy Statement and Terms of Service: How we use your data

5. Copilot cloud agent: Plan first, then code

Copilot cloud agent can now generate an implementation plan before writing any code — giving teams a review checkpoint before autonomous work begins. It can also work on a branch without opening a pull request and conduct deep codebase research sessions. Copilot Business and Enterprise admins must enable the agent to unlock these capabilities.

→ Changelog: Research, plan, and code with Copilot cloud agent

6. Model lifecycle management: New GAs, LTS designation, and model retirement

GPT-5.4 and GPT-5.4 mini are now generally available across GitHub Copilot. GPT-5.3-Codex has received a long-term support designation, offering a stable option for enterprises requiring model consistency. Additionally, Gemini 3 Pro was retired on March 26 — admins should verify users and integrations are updated to Gemini 3.1 Pro.

Sources:
→ Changelog: GPT-5.4 is generally available in GitHub Copilot
→ Changelog: GPT-5.4 mini is now generally available for GitHub Copilot
→ Changelog: GPT-5.3-Codex long-term support in GitHub Copilot
→ Changelog: Gemini 3 Pro deprecated

7. Security architecture of GitHub Agentic Workflows

GitHub published a detailed breakdown of how Agentic Workflows isolate agent execution from the broader Actions environment, constrain network access via allowlists, and provide comprehensive audit logs. This is a valuable reference for security and compliance teams evaluating agentic automation for enterprise deployment.

→ Blog: Under the hood: Security architecture of GitHub Agentic Workflows

8. Copilot SDK in public preview

The GitHub Copilot SDK is now in public preview in five languages — Node.js, Python, Go, .NET, and Java — allowing platform teams to embed Copilot's agentic runtime into internal tools and services. Enterprise customers can use Bring Your Own Key (BYOK) for custom model routing without consuming Copilot quota.
