🏆 Top 8 updates of the month

1. Enterprise agent monitoring gains granular session filters

The agent control plane in GitHub Enterprise AI Controls now lets you filter agent sessions by status, repository, and user — in addition to existing agent and organization filters. Admins gain real-time visibility into who is running agents, where, and how sessions are completing.

→ Changelog: Discover and manage agent activity with new session filters

2. New admin control: Skip workflow approval for Copilot coding agent

Repository administrators can now configure a setting to let GitHub Actions workflows run automatically when Copilot coding agent pushes changes, removing the human approval step. The default remains unchanged — approval is still required unless explicitly disabled.

→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows

3. GPT-5.4 is generally available — admins must enable the model policy

OpenAI's latest agentic coding model is now available across all Copilot surfaces. Enterprise and Business administrators must enable the GPT-5.4 policy in Copilot settings before users can select it in their model picker.

→ Changelog: GPT-5.4 is generally available in GitHub Copilot

4. Copilot code review at scale: 60M reviews, CLI integration, and 12K+ orgs

Copilot code review has surpassed 60 million reviews, now accounting for one in five code reviews on GitHub. Over 12,000 organizations run it automatically on every pull request. Developers can also now request Copilot reviews directly from the GitHub CLI.

Sources:
→ Blog: 60 million Copilot code reviews and counting
→ Changelog: Request Copilot code review from GitHub CLI

5. Agent hooks: Enforce policies during agent sessions across IDEs

Agent hooks let you run custom commands at key points during agent sessions — ideal for enforcing policies, auto-linting, or blocking commands. Now available in VS Code and JetBrains (preview). Business and Enterprise admins must enable the Editor preview features policy.

Sources:
→ Changelog: GitHub Copilot in Visual Studio Code v1.110 – February release
→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs

6. Auto model selection GA — respects admin policies with premium request savings

Auto model selection is now GA in JetBrains IDEs. Copilot dynamically picks the best available model while honoring all administrator model policies. Paid subscribers get a 10% discount on the premium request multiplier when using auto.

→ Changelog: Copilot auto model selection is generally available in JetBrains IDEs

7. Security architecture of Agentic Workflows: Zero-trust agents by design

GitHub's deep dive into Agentic Workflow security reveals a layered architecture: agents run with zero access to secrets, network traffic is firewalled to an allowlist, all GitHub writes are staged and vetted through safe outputs, and comprehensive logging supports audit and forensics.

→ Blog: Under the hood: Security architecture of GitHub Agentic Workflows

8. IDE agent maturity: Custom agents GA, instruction files, and AI coauthor attribution

Across VS Code and JetBrains, custom agents, sub-agents, and plan agent are now GA. Agent instruction files (AGENTS.md, CLAUDE.md) are supported, terminal sandboxing constrains agent execution, and AI coauthor attribution tags commits containing AI-generated code.

Sources:
→ Changelog: GitHub Copilot in Visual Studio Code v1.110 – February release
→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs
