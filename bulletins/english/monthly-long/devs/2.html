Monthly (Long) #2 · April 2026 · Devs Edition

🎯 This edition's thread

March and early April signal a clear shift in how Copilot integrates into developer workflows. The Copilot cloud agent has evolved from a pull-request generator into a full research-and-planning collaborator — capable of deep codebase exploration and step-by-step implementation guidance before writing a single line of code. At the same time, the developer toolkit has expanded significantly: a new SDK brings Copilot's agent runtime into your own applications, /fleet in Copilot CLI enables parallel subagent execution, and IDE integrations have matured across JetBrains, Visual Studio, and VS Code.

⚠️ Heads up

  • GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated — Action required — These models were retired on April 1, 2026. Switch to GPT-5.3-Codex in all workflows, scripts, and integrations. See supported models for alternatives.
  • Upcoming deprecation of Claude Sonnet 4 in GitHub Copilot — Deadline: May 1, 2026 — Migrate to Claude Sonnet 4.6 before this date. Update any workflow or integration using Claude Sonnet 4. Enterprise admins may need to enable the new model in Copilot settings.
  • Gemini 3 Pro deprecated — Action required — Deprecated on March 26, 2026. Switch to Gemini 3.1 Pro. Enterprise admins may need to enable it in model policies in Copilot settings.

🚀 Take action

  • Explore Copilot cloud agent's new research and planning workflow — ask for an implementation plan before any code is written, then iterate on a branch.
  • Build with the Copilot SDK in public preview — embed the same agent runtime powering cloud agent and Copilot CLI directly into your applications.
  • Try /fleet in Copilot CLI to run multiple agents in parallel — ideal for multi-file refactors, doc generation, and large-scope tasks.
  • If you use Claude Sonnet 4, switch to Claude Sonnet 4.6 before the May 1 deadline to avoid disruption.
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 8 updates of the month

1. Copilot cloud agent can now research, plan, and code — without opening a PR

Copilot cloud agent can now research your codebase, generate an implementation plan for review, and push changes to a branch — all before you decide to open a PR. Review the diff, iterate with the agent, and create a pull request only when you're ready.

→ Changelog: Research, plan, and code with Copilot cloud agent

2. Copilot SDK in public preview: the agent runtime, in your hands

The Copilot SDK is now available in public preview across five languages: Node.js, Python, Go, .NET, and Java. Built on the same agent runtime as cloud agent and Copilot CLI, it supports custom tools, streaming responses, BYOK, and OpenTelemetry out of the box.

→ Changelog: Copilot SDK in public preview

3. /fleet in Copilot CLI: run multiple agents in parallel

The new /fleet command in Copilot CLI dispatches multiple subagents simultaneously. Describe a multi-file objective, and an orchestrator decomposes it into parallel work items — each running independently. Ideal for large refactors, documentation, or migrations spanning many files at once.

→ Blog: Run multiple agents at once with /fleet in Copilot CLI

4. Cloud agent is 50% faster to start, smarter at code search, and fully traceable

Cloud agent now starts work 50% faster and uses semantic code search to navigate large codebases more intelligently. Every agent commit is also now traceable back to its full session log, giving you complete visibility into what the agent explored and changed.

Sources:
→ Changelog: Copilot coding agent now starts work 50% faster
→ Changelog: Copilot coding agent works faster with semantic code search
→ Changelog: Trace any Copilot coding agent commit to its session logs

5. New models: GPT-5.4 mini is generally available, Gemini 3.1 Pro expands to more IDEs

GPT-5.4 mini — OpenAI's fastest agentic model — is now generally available across all Copilot IDEs, delivering the fastest time to first token and strong codebase exploration skills. Gemini 3.1 Pro also lands in JetBrains, Xcode, and Eclipse, expanding model choice beyond VS Code.

Sources:
→ Changelog: GPT-5.4 mini is now generally available for GitHub Copilot
→ Changelog: Gemini 3.1 Pro is now available in JetBrains IDEs, Xcode, and Eclipse

6. @copilot can now resolve merge conflicts and iterate on any pull request

Mention @copilot in a pull request to resolve merge conflicts or request targeted code changes to any existing PR. These two additions remove two of the most common friction points in the review and iteration cycle.

Sources:
→ Changelog: Ask @copilot to resolve merge conflicts on pull requests
→ Changelog: Ask @copilot to make changes to a pull request

7. JetBrains gets major agentic upgrades, VS Code and Visual Studio ship monthly updates

JetBrains IDEs now feature custom agents, sub-agents, plan agent, agent hooks (preview), and MCP auto-approve — all generally available or in preview — alongside auto model selection going GA. VS Code v1.110 and Visual Studio March update also shipped this month with new improvements.

8. Organization custom instructions are now generally available

Org-level custom instructions for Copilot are now generally available. Organization admins can define natural language instructions that shape Copilot's behavior across all repositories — applied consistently in Copilot Chat, code review, and cloud agent. Configure them in your organization's Copilot settings.
