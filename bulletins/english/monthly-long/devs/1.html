Monthly (Long) #1 · March 2026 · Devs Edition

🎯 This edition's thread

This month, GitHub Copilot expanded its reach across every surface developers work on. The Copilot CLI reached general availability, bringing agentic coding to the terminal. Code review shifted to an agentic architecture with deeper repository understanding, while the coding agent gained new integrations and platform support. A wave of new models — GPT-5.4, Claude Sonnet 4.6, Gemini 3.1 Pro — gives developers more choice and capability. The theme is clear: Copilot is becoming a persistent, context-aware collaborator that works wherever you do.

⚠️ Heads up

  • Upcoming deprecation of Gemini 3 Pro and GPT-5.1 models — Deadline: March 26 (Gemini 3 Pro) and April 1 (GPT-5.1 family) — Switch to Gemini 3.1 Pro or GPT-5.3-Codex and update any model references in your workflows. Check available models for alternatives.

🚀 Take action

  • Install the Copilot CLI and explore agentic development in your terminal.
  • Try the new agentic code review on your next pull request — request it from the CLI with gh pr edit --add-reviewer @copilot``.
  • Select GPT-5.4 in the model picker to test OpenAI's latest agentic coding model across all supported clients.
  • Connect the Figma MCP server to pull design context into code and push rendered UI back to Figma.
  • Questions or need support? Reach out to your GitHub team, we're here to help.

🏆 Top 8 updates of the month

1. GitHub Copilot CLI reaches general availability

The terminal-native coding agent is now GA for all Copilot subscribers. It includes plan mode for structured task execution, autopilot mode for autonomous workflows, MCP server support, plugins, repository memory, and built-in specialized agents for code review, exploration, and testing — all without leaving your terminal.

Sources:
→ Changelog: GitHub Copilot CLI is now generally available
→ Blog: From idea to pull request: A practical guide to building with GitHub Copilot CLI

2. Expanded model lineup: GPT-5.4, Claude Sonnet 4.6, and more

GPT-5.4 is now GA across all Copilot clients, delivering enhanced agentic reasoning. Claude Sonnet 4.6 went GA, Claude and Codex models expanded to Business and Pro plans, Gemini 3.1 Pro entered preview, and Grok Code Fast 1 joined the free-tier auto selection. More models, more choice for every task.

3. Copilot code review goes agentic

Code review now runs on an agentic architecture that retrieves broader repository context, catches issues as it reads, maintains memory across reviews, and clusters related feedback. Usage has grown 10X since launch, now accounting for one in five code reviews on GitHub. You can also request Copilot code review directly from the GitHub CLI.

Sources:
→ Changelog: Copilot code review now runs on an agentic architecture
→ Blog: 60 million Copilot code reviews and counting
→ Changelog: Request Copilot code review from GitHub CLI

4. Coding agent: Jira integration, Windows support, and faster iteration

The coding agent gained a Jira integration (assign Jira issues to Copilot for AI-generated draft PRs), Windows project support, code referencing, image uploads in sessions, and a new option to skip Actions workflow approval for faster feedback loops. You can now delegate tasks from Visual Studio and assign issues from Raycast.

Sources:
→ Changelog: GitHub Copilot coding agent for Jira is now in public preview
→ Changelog: Optionally skip approval for Copilot coding agent Actions workflows
→ Changelog: Use Copilot coding agent with Windows projects

5. Major agentic capabilities in JetBrains IDEs

Custom agents, sub-agents, and plan agent are now GA in JetBrains IDEs. This update also brings agent hooks (preview) for running custom commands at key points during sessions, auto-approve for MCP servers, AGENTS.md instruction file support, a /memory command, and auto model selection GA with a 10% premium request discount.

→ Changelog: Major agentic capabilities improvements in GitHub Copilot for JetBrains IDEs

6. GitHub Copilot SDK: embed agentic execution in your apps

The Copilot SDK lets you embed the same planning and execution engine that powers Copilot CLI directly into your applications. Delegate multi-step work to agents, ground execution in structured context via MCP, and run agentic workflows outside the IDE — in desktop apps, SaaS platforms, or event-driven systems.

→ Blog: The era of "AI as text" is over. Execution is the new interface.

7. Copilot Memory now on by default

Copilot Memory is now enabled by default for Pro and Pro+ users. It stores repository-level understanding — coding conventions, architectural patterns, cross-file dependencies — and shares that knowledge across the coding agent, code review, and CLI. Memories expire after 28 days and are validated against the current codebase before use.

→ Changelog: Copilot Memory now on by default for Pro and Pro+ users in public preview

8. Copilot on the web: repository explorer, PR titles, and model picker

You can now browse a repository's file tree alongside Copilot Chat on github.com, with files automatically added as references. Copilot can generate pull request titles from your commits. And you can pick which model to use when mentioning @copilot in PR comments, giving you more control over the response quality.
